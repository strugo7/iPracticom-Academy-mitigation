| יכולת | מקור | הערה |

## 2. פריסה — שני חלונות + מגירת פרטים

- **עץ מחלקות** (ימין): היררכיה, מונה חברים, פעולות (הוסף/ערוך/סדר). בחירת
  מחלקה → הגדרותיה כולל **מנהל המחלקה**.
- **משתמשי המחלקה** (שמאל): טבלה — שם+אווטאר, אימייל, תפקיד, סטטוס, פעילות.
- **מגירת פרטי משתמש** (בלחיצה): הגדרות + פעולות.
- **הזמנות**: סקשן/לשונית לשליחה ומעקב.

## 📋 פרומפט A — מבנה ארגוני, משתמשים ושיוך מנהל (העתק מכאן)

```
אני רוצה לעצב את עמוד ניהול המשתמשים והמבנה הארגוני של iPracticom Academy
(admin בלבד, בתוך הגדרות מערכת). RTL, שפת העיצוב והטוקנים הקיימים.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— פריסה: שני חלונות —
• עץ מחלקות (ימין): היררכיה (מחלקות ותת-מחלקות, parent_id), מונה חברים לכל מחלקה.
  פעולות: "הוסף מחלקה/תת-מחלקה", ערוך, סדר מחדש (drag).
  בחירת מחלקה → פאנל הגדרות מחלקה: שם, מחלקת-אב, תיאור, ו**"מנהל המחלקה"**
  (בורר משתמש — שיוכו מגדיר את managed_department שלו ופותח לו את נתוני המחלקה).
• משתמשי המחלקה הנבחרת (שמאל): טבלה — שם+אווטאר, אימייל, תפקיד
  (משתמש/מדריך/מנהל/אדמין), סטטוס (פעיל/לא-פעיל), פעילות אחרונה.
  שורה לחיצה → מגירת פרטי משתמש.
• כפתורים עליונים: "הזמן משתמש", "מחלקה חדשה".

— הדגש על "מנהל המחלקה": כשממנים מנהל, הצג חיווי שזה מעניק לו גישה לנתוני
  עובדי המחלקה (דשבורד ניהול).

— נתוני דמו: עץ — הנהלה → (תפעול, תמיכה טכנית → טכנאי שטח · מוקד תמיכה, מכירות).
  מחלקת "טכנאי שטח" נבחרת, 4 עובדים, ומנהל מחלקה משויך.

— מצבים: דסקטופ-first + מובייל (העץ כ-drawer).
— טכני: React + Tailwind + shadcn/ui, dnd-kit (עץ), lucide-react. שמור על רכיבים.

תתחיל מהמבט המלא: עץ + משתמשי מחלקה נבחרת + פאנל הגדרות מחלקה עם שיוך מנהל.
```

## 📋 פרומפט B — מגירת משתמש (פעולות) + הזמנות (העתק מכאן)

```
עכשיו עצב את מגירת פרטי המשתמש ואת זרימת ההזמנות. אותה שפת עיצוב, RTL.

— מגירת פרטי משתמש (בלחיצה על משתמש) —
• הגדרות: שם מלא, אימייל, תפקיד (בורר), מחלקה (בורר), סטטוס (פעיל/לא-פעיל).
• פעולות:
  - "שלח מבחן כניסה": בורר מבחן (מתוך מבחני is_entrance_exam) → שליחה + התראה.
  - "שלח הודעה": מחבר הודעה קצרה → נשלחת כהתראה במערכת.
  - "שלח שוב הזמנה" / "השבת משתמש".
• מבט-מהיר: התקדמות, תעודות, פעילות אחרונה (תקציר).

— זרימת הזמנה (מודאל "הזמן משתמש") —
• טופס: אימייל, שם מלא, מחלקה (מאוכלסת מהנבחרת), תפקיד.
• יצירה → מציג magic link להעתקה + חיווי "נשלח למייל".
• סקשן "הזמנות ממתינות": רשימה עם סטטוס (ממתינה/פגה/הושלמה), מזמין,
  תאריך תפוגה, ופעולות "שלח שוב" / "בטל".

— הצג: מגירת משתמש פתוחה עם פעולות, ומודאל ההזמנה.
— טכני: React + Tailwind + shadcn. שמור על רכיבים קיימים.
```

## אנטומיה — רפרנס שדות

| אלמנט | שדה |
|---|---|
| מחלקה | `Department.{name, parent_id, order_index, description}` |
| מנהל מחלקה | `User.managed_department` (= המחלקה שהוא מנהל) |
| משתמש | `User.{full_name, email, system_role, department, status, last_activity}` |
| הזמנה | `generateInviteToken{email, full_name, department, role}` → `{magic_link, expires_at}` (טוקן hashed) |
| מבחן כניסה | `Exam.is_entrance_exam` → הקצאה + `Notification` |
| הודעה | `Notification.{user_id, type, title, message}` |
