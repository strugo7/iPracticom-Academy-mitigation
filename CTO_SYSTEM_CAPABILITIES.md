> מסמך שלישי בסדרה: מפרט את היכולות המיוחדות והלוגיקה המורכבת שנבנתה במערכת iPracticom Academy.
> משלים את `CTO_BASE44_DEPENDENCIES.md` (תלויות) ואת `BACKEND_MIGRATION_ARCHITECTURE.md` (ארכיטקטורת יעד).
> **מטרה:** שה-CTO יבין את ה"מה" לפני ה"איך" — כל יכולת עם רמת המורכבות שלה ומה קריטי לשמר.
> **תאריך:** יולי 2026

## 1. מערכת הלמידה (LMS) — הליבה

### 1.1 היררכיית תוכן בת 4 רמות
```
LearningTrack (מסלול) → TrackModule ⟶ SharedModule (מודול משותף) → Topic (נושא) → ModuleLesson (שיעור)
```
**הנקודה המתוחכמת:** מודול הוא **משותף** — אותו מודול יכול להופיע בכמה מסלולים (טבלת קישור TrackModule עם order_index). שינוי בשיעור מתעדכן בכל המסלולים. זה עיצוב מכוון שחוסך כפילות תוכן — חובה לשמר.

### 1.2 עורך השיעורים — 27 סוגי בלוקים
שיעור = מערך בלוקים ב-JSON (`blocks`), כל בלוק עם `type`, `data`, `styling`, `visibility`:
- **תוכן:** טקסט עשיר (Tiptap), כותרות, רשימות, ציטוטים, הערות, מוטיבציה, טבלאות, סקשנים מעוצבים עם ספריית תבניות
- **מדיה:** תמונות (כולל **עורך אנוטציות** — כיתובים על גבי תמונה), וידאו, PDF inline, GIF
- **אינטראקטיבי:** flashcards, quiz משובץ, labeled graphic (נקודות חמות על תמונה), tabs, **סימולטור רשת** (קנבס גרירת ציוד — מתגים, ראוטרים עם מודל קונפיגורציה), **סימולטור PBX**
- **הטמעות:** HTML embed (עם מצב picker אינטראקטיבי), Gamma, widgets
- **מערכת:** page breaks (עימוד), מפרידים, עמוד שער

**יכולות עורך מתקדמות שנבנו:** גרירה ושחרור לסידור, היסטוריית undo/redo, **גרסאות אוטומטיות** (LessonVersion — snapshot בכל שמירה + שחזור), עוזר AI פר-בלוק, ניווט TOC, תצוגה מקדימה. שני פורמטים חיים במקביל (v1 = pages, v2 = blocks) עם תאימות לאחור.

### 1.3 מערכת מושגים (Concepts)
מילון מונחים ארגוני שמוטמע **בתוך הטקסט** בכל המערכת: מרקרים מיוחדים `⟦conceptId⟧term⟦/c⟧` בתוכן, רנדור כ-tooltip/popover בלחיצה, תמיכה גם בשיעורים וגם בתסריטי תקלות, מנגנון מיגרציה למרקרים. יש גם מרקרי **סיסמאות מוסתרות** `⟦pwd⟧...⟦/pwd⟧` (הצגה ברמת הרשאה).

### 1.4 מעקב התקדמות וגיימיפיקציה
- UserProgress עם 7 סוגי אירועים (lesson_started/completed, exam_attempt/passed, module/topic/track_completed) + אחוזים, זמן, ציונים
- נעילת שיעורים סדרתית (`require_previous_lesson`), XP פר שיעור, **לוח מובילים** (getLeaderboard), חישוב סטטיסטיקות לילי (recalculateUserStats)
- **תעודות:** תבניות מעוצבות (CertificateTemplate) → הנפקה אוטומטית בסיום + מייל + מספר תעודה ייחודי + צפייה/הורדה כתמונה (html2canvas)
- **מדדי יעד למסלול** (course_metrics): יעדי השלמה/ציון/זמן לדוחות מנהלים

## 2. מערכת המבחנים — מנוע הערכה מלא

- **מאגר שאלות מרכזי** (Question) נפרד מהמבחנים — שאלה אחת משמשת כמה מבחנים, כולל שאלות סדר-פעולות (order sequence)
- מבחן = רשימת מזהי שאלות + ניקוד פר שאלה, עם: הגבלת זמן, ציון עובר, מגבלת נסיונות, ערבוב שאלות/תשובות, מדיניות פידבק (מיידי/ללא), הצגת תוצאות מותנית
- **5 סוגי הקשר:** מבחן מסלול / מודול / נושא / שיעור / עצמאי + **מבחני כניסה למועמדים** עם target_roles/departments
- ניתוח ביצועים: ExamAttempt/ExamResult, ציון ממוצע, דוחות פר מחלקה
- **QuizMaster Agent:** קורא תוכן שיעור אמיתי ומחולל מבחן שלם אוטומטית (כולל דרך WhatsApp)

## 3. גיוס והערכת מועמדים — flow עצמאי מלא 🟢

זרימה שלמה שכבר לא תלויה ב-Base44 (JWT משלנו):
1. מנהל מזמין מועמד → `generateInviteToken` יוצר **magic link** חתום (INVITE_JWT_SECRET, תפוגה)
2. מועמד נכנס בלי חשבון → עמוד ציבורי טוען מבחן דרך `fetchExamDataForCandidate` (אימות token בלבד)
3. הגשה → `submitCandidateAssessment` → ציון אוטומטי → התראה למנהלים
4. החלטה → `makeCandidateDecision` (קבלה/דחייה) → מייל + במקרה קבלה הזמנה למערכת
+ ניהול הזמנות מלא: שליחה חוזרת, ביטול, פקיעה אוטומטית (cron), **מבחני כניסה פנימיים** לעובדים חדשים לפי תפקיד/מחלקה.

## 4. ניהול ידע (KMS)

### 4.1 מנוע חיפוש ארגוני — נבנה מאפס 🟢
זו יכולת הנדסית משמעותית שכולה קוד שלנו:
- **אינדקס TF-IDF** (SearchIndex) שמתעדכן אוטומטית בכל שינוי מסמך (indexDocument/onDocumentChange/reindexAll)
- **חיפוש fuzzy** לטיפול בשגיאות כתיב + **השלמה אוטומטית** (searchAutocomplete)
- **מילים נרדפות** (SynonymGroup + expandQueryWithSynonyms) — "מרכזיה"="PBX"
- **אנליטיקת חיפוש** (SearchLog): מה מחפשים, כמה typos, מה לא נמצא — דשבורד ייעודי
- חיפוש גלובלי חוצה-ישויות (שיעורים, מאמרים, מושגים, תסריטים)

### 4.2 נהלים עם אישור קריאה (Compliance)
- נוהל (HTML עשיר או קובץ) → פרסום לפי מחלקות → **פופאפ חוסם** בכניסה למערכת עד אישור
- ProcedureAcknowledgement: מי אישר, מתי, מאיזה IP — ראיה משפטית
- דשבורד מעקב למנהלים: מי טרם אישר, פר נוהל ופר מחלקה + התראות אוטומטיות

## 5. פתרון תקלות (Troubleshooting) — מערכת בפני עצמה

- **עורך תסריטים ויזואלי** (FlowEditor על React Flow): גרף צמתים — שאלה/פעולה/פתרון/סיום/קישור-לתסריט-אחר, עם תוכן עשיר בכל צומת, גרסאות, הרשאות, ייצוא/ייבוא, ו**סוכן AI שמנתח ומתקן את לוגיקת הגרף** (מוצא צמתים יתומים, מסלולים ללא סיום)
- **נגן תסריטים** לנציגי תמיכה: צעד-אחר-צעד בזמן שיחה עם לקוח, כולל טיימר
- **סיכום CRM אוטומטי:** בסיום שיחה — `summarizeFlow` הופך את יומן הצעדים לסיכום מקצועי מוכן להדבקה ב-CRM (LLM עם JSON schema)
- **לולאת שיפור:** TroubleshootingSession מתעד שיחות שבהן **חסר תסריט** → תור "תסריטים חסרים" למנהלים → יצירת תסריט חדש
- `webSearchOem`: חיפוש חי במסמכי יצרן מאושרים בלבד (whitelist של ~20 דומיינים: Cisco, MikroTik, Hikvision...)

## 6. צינורות יצירת תוכן AI — המורכבים ביותר

### 6.1 מסמך → שיעורים (AILessonJob pipeline)
העלאת PDF/מסמך → עיבוד רב-שלבי עם מצב מתועד (`pipeline_state`, progress %): חילוץ תוכן כולל **תמונות עם callouts וקישור לטקסט**, יצירת מבנה שיעור (מטרות, סקשנים, סיכום, quiz) → טיוטה לאישור עורך → פרסום להיררכיה. חלק רץ ב-**n8n** עם callbacks חתומים HMAC + התראות בסיום/כשל.

### 6.2 יצירת שיעור in-house (אשף רב-שלבי)
InHouseLessonWizard: הגדרות → מיקום בהיררכיה → יצירת תוכן (Claude ישירות, פרומפט פדגוגי בעברית שמחזיר **בלוקים מובנים ב-JSON**) → כולל **חיפוש בידע הפנימי** (RAG בסיסי — שליפת מאמרים/שיעורים רלוונטיים והזרקתם לפרומפט) + ולידציה ותיקון אוטומטי של מבנה הבלוקים.

### 6.3 אינטגרציות תוכן נוספות
- **Gamma:** יצירת מצגות מתוכן שיעור דרך n8n + callback
- **Flipbooks** דרך n8n
- **Google Slides/Docs:** ייבוא מצגות (עם thumbnails ושיוך להיררכיה) ויצירת מסמכים — דרך OAuth
- **יצירת תמונות AI** (OpenRouter עם preset מותאם) + עיבוד PDF מאובטח (Adobe)

### 6.4 חמשת סוכני ה-AI
eduSupport (תמיכה דו-מצבית עם ציטוט מקורות + טיקטים), quizMaster (+WhatsApp), flow_builder, lesson_pdf_generator, changelog_manager — פירוט ב-`CTO_BASE44_DEPENDENCIES.md` סעיף 5.

## 7. אבטחה — שכבות שנבנו מעל ה-auth הבסיסי

- **2FA במייל:** OTP בן 6 ספרות עם תפוגה (UserOtp) + rate limiting
- **Whitelist דומיינים + טווחי IP** (checkIpAccess כ-gate)
- **SecurityLog:** כל ניסיון חסום/כניסה מתועד (IP, UA, סוג)
- **סורק אבטחה פנימי** (runSecurityAudit/runAuditScan) שמאתר ממצאים + **ניתוח AI לכל ממצא** עם צעדי תיקון
- **מצב "View As":** אדמין מדמה תפקיד אחר לבדיקת הרשאות (simulatedRole)
- **4 תפקידים** (admin/manager/instructor/user) + מנהלי מחלקה רואים רק את המחלקה שלהם, RLS על ~20 ישויות

## 8. ניהול ותפעול

- **דשבורד מנהלים:** אנליטיקת מחלקות, עובדים "תקועים" (14+ ימים ללא פעילות), דוחות השלמה/ביצועי מבחנים/פידבק, שליחת הודעות והקצאת מסלולים, **מבנה ארגוני היררכי** (Department עם parent_id)
- **התראות:** 14 סוגים, עדיפויות, **dedupe_key** למניעת כפילויות, פעמון + מיילים (Resend)
- **אשפי הדרכה (Wizards):** סיורי מערכת מוגדרים דינמית (WizardConfig) עם עורך ויזואלי לבחירת אלמנטים בעמוד
- **Changelog:** יומן עדכוני מערכת עם טיימליין/לוח שנה, מדיה, טופס פידבק + סוכן AI לכתיבה
- **שעון נוכחות** + ייצוא שעות עבודה
- **ייצוא/ייבוא:** שיעור כ-HTML+ZIP עצמאי, PDF, CSV לשאלות/מושגים, **גיבוי JSON מלא של כל המערכת** כולל משתמשים (exportAllData/importAllData) — זה גם קובץ ה-seed למסד החדש
- **Schema Explorer:** כלי פנימי שמתעד את כל הישויות, הקשרים והפונקציות ויזואלית — כבר קיים כתיעוד חי למיגרציה

## 9. סיכום ל-CTO — מדד מורכבות שחזור

| יכולת | מורכבות שחזור | הערה |
|---|---|---|
| היררכיית למידה + מודולים משותפים | בינונית | דאטה-מודל מוכן, CRUD |
| עורך 27 בלוקים + גרסאות | **גבוהה** | הרבה UI קיים 🟢, ה-backend רק שומר JSON |
| מנוע חיפוש | נמוכה | קוד נייד 🟢, או החלפה ב-Postgres FTS |
| מבחנים + מועמדים | בינונית | ה-magic-link כבר עצמאי 🟢 |
| תסריטי תקלות | בינונית | UI קיים 🟢, backend = CRUD + LLM לסיכום |
| צינורות AI | **גבוהה** | n8n נשאר 🟢, נדרש LLM gateway + queue |
| סוכני AI + WhatsApp | **הגבוהה ביותר** | runtime שלם לבנות |
| אבטחה (2FA, whitelist, audit) | נמוכה | קוד נייד ברובו 🟢 |
| נהלים + compliance | נמוכה | CRUD + event handlers |
