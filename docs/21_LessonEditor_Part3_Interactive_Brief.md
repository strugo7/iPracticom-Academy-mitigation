## 2. כרטיסי זיכרון (flashcard)

`data = {cards:[{front, back}]}`.
- **תצוגת לומד:** כרטיס שמתהפך (flip) בלחיצה — צד קדמי (שאלה/מונח) → אחורי
  (תשובה/הסבר). ניווט בין כרטיסים (הקודם/הבא + מונה).
- **עריכה:** רשימת כרטיסים, כל אחד עם שדה קדמי ואחורי; הוסף/מחק/סדר (גרירה).
  אפשרות מדיה בכל צד (Media Picker).

## 3. טאבים (tabs)

`data = {tabs:[{title, content}]}`.
- **תצוגת לומד:** שורת לשוניות; לחיצה מחליפה תוכן.
- **עריכה:** הוסף/מחק/סדר לשוניות; כל לשונית עם כותרת ותוכן (תוכן יכול להכיל
  בלוקים מקוננים פשוטים — לפחות טקסט/מדיה).

---

## 4. טופולוגיית רשת (network_canvas) — עורך-גרף ייעודי

**מהות (לפי המפרט שלך):** מדריך/אדמין בונה טופולוגיה של רכיבי רשת עם הגדרות
וחיבורים — לצורך הדגמה/תרגול בשיעור.

### רכיבי הרשת (סוגי צמתים)
ראוטר · מודם · סוויץ' · אקסס פוינט · iPad · טלפון VoIP · מדפסת/קופה · קופה
רושמת · בולק אשראי · ועוד. לכל רכיב אייקון ייעודי + הגדרות רשת (IP, שם, פורט...).

### העורך (תאום FlowEditor — מסמך 06)
- **קנבס** עם רכיבים ממוקמים + חיבורים (קווי רשת) ביניהם.
- **פלטת רכיבים:** גרירת סוגי הרכיבים לקנבס, צבע/אייקון לכל סוג.
- **Inspector לרכיב:** שם, סוג, הגדרות רשת (כתובת IP, subnet, פורטים...).
- **חיבורים:** גרירה מרכיב לרכיב; כל חיבור = קו רשת (אפשר עם תווית פורט).
- **פאנל ניהול הסוויץ' (ייחודי):** בבחירת סוויץ' — טבלת פורטים שמראה איזה רכיב
  מחובר לאיזה פורט (port → device), כמו ממשק ניהול סוויץ' אמיתי בקטן.

### תצוגת לומד
טופולוגיה אינטראקטיבית: לחיצה על רכיב → הגדרותיו; צפייה בפאנל הסוויץ'.
(אפשר מצב קריאה-בלבד או תרגול.)

## 📋 פרומפט A — כרטיסי זיכרון וטאבים (העתק מכאן)

```
אני רוצה לעצב שני בלוקים אינטראקטיביים לעורך השיעורים של iPracticom Academy
(חלק 3, על המעטפת הקיימת). RTL, אותה שפת עיצוב וטוקנים.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

1) כרטיסי זיכרון (flashcard):
   • תצוגת לומד: כרטיס שמתהפך באנימציה בלחיצה — צד קדמי (מונח/שאלה) → אחורי
     (הסבר/תשובה). ניווט הקודם/הבא + מונה כרטיסים.
   • עריכה: רשימת כרטיסים, כל אחד עם שדה קדמי ואחורי + אפשרות מדיה (Media Picker);
     הוסף/מחק/סדר בגרירה.
   • דמו: 3 כרטיסים על מונחי רשת (VLAN, DHCP, PoE).

2) טאבים (tabs):
   • תצוגת לומד: שורת לשוניות; לחיצה מחליפה תוכן בהחלקה עדינה.
   • עריכה: הוסף/מחק/סדר לשוניות; כל לשונית עם כותרת ותוכן.
   • דמו: 3 לשוניות ("הגדרה", "דוגמה", "טיפים").

— הצג את שניהם גם במצב תצוגת-לומד וגם במצב עריכה.
— טכני: React + Tailwind + shadcn/ui, lucide-react, framer-motion (flip/slide),
  dnd-kit (סידור). נגישות: ניווט מקלדת בכרטיסים ובלשוניות, ARIA. שמור על רכיבים.

תתחיל מכרטיס הזיכרון בתצוגת לומד (עם אנימציית ההיפוך).
```

## 📋 פרומפט B — טופולוגיית רשת (network_canvas) (העתק מכאן)

```
אני רוצה לעצב את בלוק טופולוגיית הרשת (network_canvas) לעורך השיעורים של
iPracticom Academy. זהו עורך-גרף — בנה אותו על אותם עקרונות של עורך ה-Playbooks
(FlowEditor): custom nodes, חיבורים, ו-Inspector. RTL, אותה שפת עיצוב.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 · אייקוני outline.

— מהות: מדריך בונה טופולוגיית רשת — רכיבים, הגדרות, וחיבורים — להדגמה בשיעור.

— רכיבי הרשת (צמתים, כל אחד עם אייקון ייעודי) —
ראוטר · מודם · סוויץ' · אקסס פוינט · iPad · טלפון VoIP · מדפסת/קופה ·
קופה רושמת · בולק אשראי. (פלטה לגרירה לקנבס.)

— העורך —
• קנבס עם רכיבים ממוקמים + חיבורי רשת (קווים) ביניהם; רקע נקודות, זום, minimap.
• Inspector לרכיב נבחר: שם, סוג, והגדרות רשת (כתובת IP, subnet, פורט).
• חיבורים: גרירה מרכיב לרכיב; תווית פורט אופציונלית על החיבור.
• פאנל ניהול סוויץ' (ייחודי וחשוב): בבחירת סוויץ' — טבלת פורטים שמציגה
  port → device (איזה רכיב מחובר לאיזה פורט), כמו ממשק ניהול סוויץ' מוקטן.

— תצוגת לומד —
טופולוגיה אינטראקטיבית: לחיצה על רכיב → הגדרותיו; צפייה בפאנל הסוויץ'.

— דמו: רשת קטנה — מודם → ראוטר → סוויץ' → (אקסס פוינט, טלפון VoIP, קופה,
  בולק אשראי). בחר את הסוויץ' והצג את טבלת הפורטים שלו.

— טכני: React + Tailwind + shadcn/ui, @xyflow/react (כמו FlowEditor),
  lucide-react. החיווט החי (גרירה/חיבורים) יחובר ב-Claude Code; עצב את הרכיבים,
  ה-Inspector, ופאנל הסוויץ'. שמור על רכיבים קיימים.

תתחיל מהקנבס עם טופולוגיית הדמו והסוויץ' נבחר (פאנל הפורטים פתוח).
```

## אנטומיה — רפרנס שדות

| בלוק | `data` |
|---|---|
| flashcard | `{cards:[{front, back, media?}]}` |
| tabs | `{tabs:[{title, content}]}` |
| network_canvas | `{nodes:[{id, type, label, settings{ip,subnet,ports}, position}], connections:[{source, target, sourcePort, targetPort, label}]}` *(מבנה מוצע — לאמת/להגדיר)* |

> network_canvas: מבנה ה-`data` לא מפורט ב-SRS — המבנה לעיל מוצע לפי המפרט שלך.
> לאמת/לקבע בקוד.
