# בריף עיצוב — נגן ה-Playbooks (תצוגת מחשב) + תיעוד סשן + סיכום AI

> **מטרה:** גרסת דסקטופ של ה-FlowPlayer, עם תיעוד מלא של הסשן, סיכום AI, ומסלול "לא נפתר".
> **תפקיד (PRD §11):** `FlowPlayer` — תצוגת מחשב.
> **בונה על:** מסמך 07 (הנגן mobile-first) — אותם צמתים, אותה שפת צבעים. כאן נוסף מרחב הדסקטופ ושלוש יכולות.
> **מקורות:** PRD §4.3, §5.5 · SRS §1.8 (FlowFeedback, TroubleshootingSession, session_log).

---

## 1. פריסת הדסקטופ — שתי חטיבות

המרחב הנוסף של המחשב מאפשר להציג את **תיעוד הסשן באופן קבוע** לצד הצומת הנוכחי:

```
┌──────────────────────────────────────────────────────────────┐
│  שם ה-Playbook    [התחל מחדש]  [התקלה לא נפתרה]  [סגור]         │
├────────────────────────────────────┬─────────────────────────┤
│                                    │  תיעוד הסשן (קבוע)        │
│   הצומת הנוכחי — גדול, ממורכז        │  ציר-זמן הצעדים:         │
│   שאלה / פעולה / פתרון               │  ● start · 14:32        │
│                                    │  ● "LED לא דולק" · 14:33 │
│   [אפשרויות גדולות / צ'קליסט]         │  ● בדיקת PoE · 14:34    │
│                                    │  ...                    │
│   פעולה ראשית                       │  [ייצוא תיעוד]          │
└────────────────────────────────────┴─────────────────────────┘
```

- **אזור ראשי** (צד ימין ב-RTL): הצומת הנוכחי, גדול וברור — כמו בנגן המובייל.
- **פאנל תיעוד** (צד שמאל, קבוע): ציר-זמן חי של הצעדים שבוצעו, מתעדכן בכל מעבר.
- במובייל הפאנל נשאר collapsible (כמו במסמך 07); במחשב הוא גלוי תמיד.

---

## 2. מסלול "התקלה לא נפתרה"

- **זמין תמיד** — כפתור משני בכותרת, נגיש בכל שלב (לא רק בסוף). טכנאי שמבין
  באמצע שה-Playbook לא עוזר יכול לצאת מסודר.
- **גם כתוצאת קצה** — בנוסף ל-`solution` (הצלחה) ול-`end` (הסלמה).
- **בלחיצה:** מסך סיכום "לא נפתר" (ענבר/אדום), עם סיכום ה-AI והתיעוד, ואפשרויות:
  • "דווח על Playbook חסר/חלקי" → כותב `missing_flow=true` +
    `missing_flow_description` → **מזין את לשונית 'תסריטים חסרים' (מסמך 05)**.
  • "פתח קריאת שירות / הסלם".
  • משוב עם `was_helpful=false`, `solution_found=false`.

> זו הלולאה שסוגרת את האשכול: כשלון אבחון הופך לאות לארגון איפה חסר Playbook.

---

## 3. תיעוד הסשן (session_log גלוי)

ה-`session_log[]` (`{nodeId, action, timestamp, details}`) שהיה שקוף — כאן הופך
לרשומה גלויה:
- ציר-זמן נקי: כל צומת שבוקר, כל בחירה שנעשתה, חותמת זמן, ומשך כולל.
- **ניתן לייצוא** (העתקה / PDF) — לצירוף לקריאת שירות.
- מתעדכן חי בכל מעבר.

---

## 4. סיכום ה-AI (בסוף — דוח הסשן)

כרטיס סיכום שנוצר ע"י AI (פונקציה חדשה `summarizeSession`) משלושה חלקים:
1. **תיאור הבעיה** — מה הייתה התקלה.
2. **הממצאים** — מה התגלה דרך שלבי האבחון.
3. **סיכום הפתרון שבוצע** — מה נעשה (או מה נוסה, אם לא נפתר).

- מצב טעינה: "ה-AI מסכם את הסשן…".
- **העתקה / ייצוא** — מוכן להדבקה בקריאת שירות או למאגר ידע.

**דמו (לעיצוב):**
> תיאור הבעיה: מצלמת אבטחה בכניסה הראשית לא הציגה תמונה ב-NVR.
> ממצאים: נורית ה-LED לא דלקה; כבל ה-PoE לא היה מחובר כראוי לפורט בסוויץ'.
> סיכום הפתרון: חובר מחדש כבל ה-PoE, המצלמה עלתה לרשת והתמונה חזרה. נפתר תוך 4 דקות.

## אנטומיה — רפרנס שדות

| אלמנט | שדה |
|---|---|
| תיעוד צעדים | `FlowFeedback.session_log[].{nodeId, action, timestamp, details}` |
| תוצאה: נפתר | `solution` node → `was_helpful=true` |
| תוצאה: לא נפתר | `was_helpful=false`, `solution_found=false` |
| דיווח חסר | `TroubleshootingSession.{missing_flow=true, missing_flow_description, handled}` |
| משך | `duration_minutes` |
| סיכום AI | פונקציה חדשה `summarizeSession` → `{problem_description, findings, solution_summary}` |

> **`summarizeSession` (חדשה):** edge function שמקבלת `session_log` + תוכן ה-Playbook
> + התוצאה, וקוראת ל-OpenRouter/Claude. שונה מ-`summarizeFlow` (שמסכמת את *הגדרת*
> ה-Playbook, לא את *הריצה*). תיבנה ב-Claude Code.
