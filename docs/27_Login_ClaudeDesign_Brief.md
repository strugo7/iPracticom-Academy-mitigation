# בריף עיצוב — מסך התחברות

> **מטרה:** מסך הכניסה למערכת — Google SSO, אימות דו-שלבי (OTP), ומצבי שגיאה.
> **תפקיד:** נקודת הכניסה, לפני המעטפת (מסמך 11).
> **מקורות:** SRS — Auth (Google OAuth), UserOtp (2FA), AppSetting (domain whitelist), SecurityLog.

---

## 1. זרימת האימות

```
התחברות עם Google → בדיקת דומיין מורשה (whitelist)
   → תקין: אימות דו-שלבי (OTP במייל) → כניסה
   → לא מורשה: מסך "גישה נדחתה"
```

> כל ניסיון נרשם ל-`SecurityLog` (success/blocked + attempt_type).

---

## 2. המסכים/מצבים

1. **התחברות ראשית** — לוגו, ברכה, "התחבר עם Google".
2. **אימות דו-שלבי (OTP)** — הזנת קוד 6-ספרות שנשלח למייל, שליחה חוזרת, טיימר.
3. **שגיאות** — דומיין לא מורשה · גישה נדחתה (לא ב-whitelist/לא הוזמן) · קוד שגוי.
4. **(נלווה)** כניסה דרך magic link מהזמנה.

```
הקיימים. זה הרושם הראשון — מקצועי, נקי, ובטוח.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— פריסה: split — פאנל מותגי בצד אחד, כרטיס ההתחברות בצד השני (ב-RTL: הטופס
  בצד ההתחלה). הפאנל המותגי: gradient #2EB4FF→#0075DB עם מוטיב עדין של
  טופולוגיית רשת (צמתים וחיבורים), לוגו, וטאגליין. בתחתית: "לשימוש פנימי בלבד".

— מסך 1: התחברות ראשית —
• לוגו iPracticom + ברכה ("ברוכים הבאים ל-iPracticom Academy").
• כפתור ראשי "התחבר עם Google" (אייקון Google).
• טקסט עזר עדין: "התחברות מאובטחת לעובדי הארגון".

— מסך 2: אימות דו-שלבי (OTP) —
• כותרת "אימות דו-שלבי", הסבר "שלחנו קוד למייל שלך".
• 6 תאי ספרה (OTP input).
• "שלח קוד מחדש" + טיימר ספירה-לאחור.
• כפתור "אמת והתחבר".

— מסכי שגיאה —
• דומיין לא מורשה: "התחברות אפשרית רק עם חשבון ארגוני".
• גישה נדחתה: "אין לך גישה למערכת — פנה למנהל" (לא ב-whitelist / לא הוזמן).
• קוד שגוי: הודעה עדינה ליד תאי ה-OTP.

— הצג את שלושת המסכים (התחברות, OTP, ושגיאת גישה נדחתה).
— מצבים: דסקטופ-first + מובייל (הפאנל המותגי מתכווץ לכותרת עליונה).
— טכני: React + Tailwind + shadcn/ui, lucide-react (outline), framer-motion.
  נגישות: focus על תאי OTP, ARIA, ניווט מקלדת. שמור על רכיבים קיימים.

תתחיל ממסך ההתחברות הראשי בדסקטופ.
```

## אנטומיה — רפרנס

| אלמנט | מקור |
|---|---|
| התחברות | Google OAuth (Supabase Auth) |
| אימות דו-שלבי | `UserOtp` / Email OTP |
| בדיקת דומיין | `AppSetting` (whitelist) — נאכף בשרת |
| רישום ניסיונות | `SecurityLog.{attempt_type, status}` |
| כניסה מהזמנה | magic link (consumeInvitation) |

## handoff ל-Claude Code

```
סיכום מימוש למפתח:
• Auth: Supabase Auth → Google OAuth provider.
• whitelist: בדיקת דומיין בשרת (AppSetting) לפני יצירת סשן; דחייה →
  SecurityLog(whitelist_denied) + מסך גישה נדחתה.
• 2FA: Email OTP (UserOtp / Supabase) — שליחת קוד, אימות, rate-limit + טיימר.
• כל ניסיון → SecurityLog (success/blocked, attempt_type).
• magic link מהזמנה → consumeInvitation (אימות email-match, 403 על mismatch).
• אחרי כניסה מוצלחת → המעטפת (מסמך 11) עם ניווט לפי system_role.
• נגישות: focus management ב-OTP, ARIA.
שמור על React + Tailwind + shadcn. RTL.
```
