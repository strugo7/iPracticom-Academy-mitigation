# בריף עיצוב — עמוד פרופיל אישי

> **מטרה:** עמוד פרופיל עשיר בהשראת תמונת הרפרנס, מעוגן בנתוני iPracticom.
> **תפקיד (PRD §11):** פרופיל אישי — כל המשתמשים.
> **מקורות:** SRS §1.1 (User/progress_stats), §1.4 (ExamAttempt/Question), §1.6 (UserCertificate), §1 (LessonNote).

---

## 1. מיפוי הבקשות → נתונים (מה מעוגן)

| בקשה | מקור נתונים | סטטוס |
|---|---|---|
| תמונת פרופיל | `User.profile_image_url` (לאמת/להוסיף) + העלאה לאחסון | מעוגן (שדה לאמת) |
| הכשרה משויכת + התקדמות | `assigned_track_id` + `progress_stats.avg_progress` | ✓ מעוגן |
| כל התעודות | `UserCertificate` (`certificate_number`, `issue_date`, `score`, `pdf_url`, `verification_code`) | ✓ מעוגן |
| ציוני מבחנים + פירוט שאלות + הסברים | `ExamAttempt.detailed_results.questions[]` + `Question.explanation` | ✓ מעוגן (חזק) |
| הערות | `LessonNote` | ✓ קיים (לאמת שדות בקוד) |

## 2. המלצות נוספות (מה עוד כדאי) — מסומן מעוגן / חדש

**מעוגן (נתונים קיימים — מומלץ להוסיף):**
- **בלוק סטטיסטיקות** (כמו "Learning Statistic" בתמונה): שעות למידה
  (`total_time_spent_minutes`), שיעורים/קורסים שהושלמו, תעודות, מבחנים שעברת,
  סך XP — הכול ב-`progress_stats`.
- **גיימיפיקציה:** XP + רמה (`total_xp`) + מיקום בטבלת המובילים (`getLeaderboard`).
- **היסטוריית למידה** (כמו "Learning History" בתמונה): שיעורים/מסלולים שהושלמו
  עם משך — מ-`UserProgress`.
- **רדאר ביצועים** (התאמה של ה-spider chart בתמונה): ציון ממוצע לפי נושא/מודול,
  נגזר מ-`ExamAttempt` — עושר ויזואלי מנתונים אמיתיים.
- **פעילות שבועית:** `weekly_lessons`, `weekly_time_spent_minutes`.
- **הגדרות חשבון:** שם, אימייל, סיסמה, **2FA** (`UserOtp` קיים), העדפות התראות.

**פיצ'ר חדש (דורש מודל נתונים — לעתיד):**
- **רדאר קומפטנציות מופשט** (כמו בתמונה) — דורש מודל מיומנויות. ההתאמה למעלה
  (רדאר ביצועים) נותנת אפקט דומה בלי מודל חדש.
- **תגי-התקדמות** (badges 4/5 כמו בתמונה) — דורש מודל גיימיפיקציה. כיום יש
  הישגים *שהושגו* (`UserCertificate.achievement_type`), לא התקדמות-לעבר-תג.
- **רצף למידה (streak)** — דורש מעקב ימים רצופים.

## 3. פריסת העמוד (בהשראת התמונה)

תצוגת דשבורד עם כרטיסים, אפשר עם לשוניות לחלקים הכבדים:

1) **כרטיס זהות** (ימין-עליון): תמונת פרופיל (ניתנת להעלאה) עם טבעת התקדמות
   (`avg_progress`), שם, תפקיד/מחלקה, ובמבט-אחד: XP, רמה, מספר תעודות.
2) **הכשרה משויכת + התקדמות:** המסלול שלי + בר התקדמות + "המשך".
3) **בלוק סטטיסטיקות:** שעות, קורסים, תעודות, מבחנים, XP.
4) **רדאר ביצועים:** ציון לפי נושא/מודול.
5) **גלריית תעודות:** כל ה-UserCertificates — תמונה, כותרת, תאריך, ציון, הורדה,
   קוד אימות.
6) **היסטוריית מבחנים (drill-down):** רשימת ניסיונות → הרחבה לפירוט שאלות.
7) **הערות:** רשימת LessonNotes, לחיפוש.
8) **הגדרות חשבון** (לשונית/מודאל).

## 📋 הפרומפט — העתק מכאן

```
אני רוצה לעצב עמוד פרופיל אישי ל-iPracticom Academy, בהשראת תצוגת דשבורד-פרופיל
עשירה (כרטיסים). השתמש בשפת העיצוב והטוקנים הקיימים בפרויקט — עקביות מלאה, RTL.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות · לוגו שמאל-עליון 50% · פוטר מותג.

— מבנה העמוד (כרטיסים) —

1) כרטיס זהות (בולט): תמונת פרופיל עגולה עם **כפתור העלאה/החלפה**, מוקפת טבעת
   התקדמות (אחוז). שם, תפקיד ("טכנאי שטח"), מחלקה. שורת מבט-מהיר: XP, רמה, תעודות.

2) הכשרה משויכת: כרטיס "המסלול שלי" עם בר התקדמות + כפתור "המשך ללמוד".

3) בלוק סטטיסטיקות (כמו Learning Statistic): שעות למידה, קורסים שהושלמו,
   תעודות, מבחנים שעברת, סך XP — כל אחד עם אייקון outline וערך גדול.

4) רדאר ביצועים: spider chart של ציון ממוצע לפי נושא (רשתות, Firewall, מצלמות...).

5) גלריית תעודות: רשת כרטיסי תעודה — תמונה ממוזערת, כותרת, תאריך הנפקה, ציון,
   כפתור הורדה (PDF), וקוד אימות.

6) היסטוריית מבחנים — drill-down: רשימת מבחנים (כותרת, ציון, עבר/נכשל, תאריך).
   לחיצה על מבחן מרחיבה לפירוט מלא של השאלות: לכל שאלה — נוסח השאלה, התשובה
   שבחר המשתמש (מודגשת), התשובה הנכונה (ירוק), אינדיקטור ✓/✗, ניקוד,
   ו**ההסבר לתשובה**. זה כלי למידה-מטעויות — עצב אותו ברור ומזמין.

7) הערות: רשימת ההערות שהמשתמש כתב (תוכן, השיעור המקושר, תאריך), עם חיפוש.

8) הגדרות חשבון (לשונית/מודאל): שם, אימייל, החלפת סיסמה, אימות דו-שלבי (2FA),
   העדפות התראות.

— נתוני דמו בעברית: טכנאי שטח, מסלול "הכשרת טכנאי שטח" 58%, מבחן "יסודות רשתות"
  85% (הרחב לשאלה לדוגמה עם הסבר), 2-3 תעודות, כמה הערות.

— מצבים: עצב לדסקטופ, והראה גם גרסת מובייל (כרטיסים בעמודה).
— טכני: React + Tailwind + shadcn/ui, lucide-react (outline), recharts לרדאר,
  framer-motion. שמור על רכיבים קיימים — אל תפשט/תמיר.

תתחיל מהמבט המלא בדסקטופ עם כרטיס הזהות, הסטטיסטיקות והרדאר, ואחר כך נדייק את
ה-drill-down של המבחנים.
```

## אנטומיה — רפרנס שדות

| אזור | שדות |
|---|---|
| זהות | `User.{full_name, profile_image_url, system_role, department}` + `progress_stats.{total_xp, avg_progress, certificates_earned}` |
| סטטיסטיקות | `progress_stats.{total_time_spent_minutes, lessons_completed, completed_courses, exams_passed, total_xp}` |
| תעודות | `UserCertificate.{certificate_title, issue_date, score, pdf_url, certificate_number, verification_code}` |
| מבחנים | `ExamAttempt.{score, passed, submitted_at, attempt_number, detailed_results.questions[]}` |
| פירוט שאלה | `detailed_results.questions[].{question_id, user_answer, correct_answer, is_correct, points_earned, max_points}` + join ל-`Question.{question_text, options, explanation}` |
| הערות | `LessonNote` (לאמת שדות: צפוי `user_id, lesson_id, content, created_at`) |
| רדאר ביצועים | נגזר: ממוצע `ExamAttempt.score` לפי נושא/מודול |
