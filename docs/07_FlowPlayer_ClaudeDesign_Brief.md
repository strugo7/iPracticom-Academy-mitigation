# בריף עיצוב — נגן ה-Playbooks (FlowPlayer)

> **מטרה:** לעצב את חוויית הקצה של פתרון בעיות — נגן אינטראקטיבי צעד-אחר-צעד.
> **תפקיד (PRD §11):** `FlowPlayer` — כולם. **mobile-first** (טכנאי שטח על הטלפון).
> **מקורות:** PRD §4.3, §5.5 · SRS §1.8.1, §1.8 (FlowFeedback).
> **יושב על:** שפת הצבעים לצמתים שננעלה במסמך 05.

---

## 1. עיקרון העיצוב — צעד אחד, מסך אחד

הנגן **לא מציג את הגרף**. הוא מציג את הצומת הנוכחי בלבד, בגדול וברור. המשתמש
הוא טכנאי בשטח, מול לקוח, על הטלפון — רגוע, ממוקד, מהיר, מגע. ההפך הגמור מהעורך.

- מסך אחד = צומת אחד.
- פעולה אחת מובהקת בכל רגע.
- ניווט "חזרה" תמיד זמין + שובל מסלול עדין.
- מדיה גדולה (התמונות שצורפו לאופציות בעורך — מופיעות כאן כעזר ויזואלי).

---

## 2. רינדור הצמתים בנגן (per type)

| סוג | איך נראה למשתמש |
|---|---|
| `start` | כרטיס פתיחה: כותרת, תיאור, מדיה, כפתור גדול "התחל אבחון". |
| `question` | השאלה (כותרת+תיאור+מדיה) + ה-`options[]` ככפתורים גדולים מלאי-רוחב; כל אופציה עם הטקסט שלה ו**המדיה שלה** (תמונה/GIF) אם קיימת. לחיצה → ניווט. |
| `action` | כותרת+תיאור + ה-`actions[]` כ**צ'קליסט** לסימון; כל פעולה עם טקסט, מדיה, וקישור חיצוני אם יש. כפתור "בוצע, המשך". |
| `solution` | מסך הצלחה ירוק: ✓ "הבעיה נפתרה!" + תיאור הפתרון → מעבר למשוב. |
| `end` | מסך סיום (למשל "הסלם לתמיכה") + הוראות → מעבר למשוב. |
| `linked_flow` | המשך חלק ל-Playbook אחר; כיתוב עדין "ממשיך ל: [שם]". |

> `note` של צומת/אופציה → מידע משלים בשורה משנית או tooltip "i".
> `hyperlink{url,label}` → כפתור/קישור ("פתח מדריך").

---

## 3. כרום הניווט

- **כותרת עליונה דביקה:** שם ה-Playbook + כפתור "חזרה" (צעד אחורה) + סגירה.
- **שובל מסלול** (collapsible): הבחירות שנעשו עד כה ("התחל → LED לא דולק → ...")
  — מאפשר התמצאות וקפיצה אחורה. עדין במובייל.
- **כפתור "התחל מחדש".**
- **פעולה ראשית מעוגנת לתחתית** (נגישה לאגודל במובייל).

---

## 4. טופס המשוב (`FlowFeedback`) — בסיום

מופיע אחרי `solution`/`end`:
- **האם עזר?** (כן/לא) — `was_helpful` (ראשי, חובה).
- **דירוג** 1–5 כוכבים — `rating`.
- **מצב רוח הלקוח** — `customer_sentiment` (positive/neutral/frustrated/angry)
  כפרצופי אמוג'י לבחירה.
- **הערות + הצעות** (טקסט חופשי, אופציונלי) — `feedback_text` / `suggestions`.
- **תמליץ?** — `would_recommend`.
- נתפס אוטומטית (לא מוצג): `resolved_at_step`, `step_number`, `duration_minutes`,
  ו-`session_log[]`.

```
אני רוצה לעצב את נגן ה-Playbooks (FlowPlayer) של iPracticom Academy —

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות. פוטר מותג.
— שפת צבעים לצמתים (כמו במסמכים הקודמים): start כחול-כהה · question כחול-מותג ·
  action ענבר · solution ירוק · end אפור · linked_flow סגול.

— עיקרון: צעד אחד = מסך אחד. גדול, רגוע, פעולה אחת מובהקת. ניווט "חזרה" תמיד.

— מבנה הנגן —
• כותרת עליונה דביקה: שם ה-Playbook + כפתור "חזרה" + סגירה.
• שובל מסלול עדין (collapsible) של הבחירות שנעשו.
• אזור הצומת הנוכחי (גדול, ממורכז במובייל).
• פעולה ראשית מעוגנת לתחתית.

— הצג את חמשת מצבי הנגן (כל אחד כמסך נפרד) —
1) start: כרטיס פתיחה "תקלת מצלמת אבטחה" + תיאור + כפתור "התחל אבחון".
2) question (המצב המרכזי): "האם נורית ה-LED במצלמה דולקת?" עם שתי אפשרויות
   ככפתורים גדולים מלאי-רוחב — "כן" ו-"לא" — אחת מהן עם תמונת עזר קטנה מצורפת.
3) action: "בדוק את חיבור החשמל ו-PoE" כצ'קליסט עם 3 פריטים לסימון + כפתור
   "בוצע, המשך".
4) solution: מסך הצלחה ירוק "✓ הבעיה נפתרה!" + תיאור.
5) feedback: טופס משוב — "האם עזר?" (כן/לא), דירוג כוכבים, מצב רוח הלקוח
   (4 פרצופי אמוג'י), והערות חופשיות.

— מצבים: עצב mobile-first, ואז הראה גם גרסת דסקטופ (עמודת קריאה ממורכזת).

— טכני: React + Tailwind + shadcn/ui, lucide-react (outline), framer-motion
(מעבר slide עדין בין צעדים). שמור על רכיבים קיימים — אל תפשט/תמיר.

תתחיל ממצב ה-question במובייל (המצב המרכזי), ואחר כך נדייק את השאר.
```

## אנטומיה — רפרנס שדות (SRS §1.8)

| אלמנט | שדה |
|---|---|
| צומת נוכחי | `nodes[].{type, title, description, note, media[]}` |
| אפשרויות שאלה | `options[].{text, note, targetNodeId, hyperlink, media[]}` |
| פעולות | `actions[].{text, note, hyperlink, media[]}` |
| מעבר deep-link | `options[].{linkedFlowId, linkedTargetNodeId}` |
| משוב | `FlowFeedback.{was_helpful, rating, customer_sentiment, feedback_text, suggestions, would_recommend}` |
| נתפס אוטומטית | `resolved_at_step, step_number, duration_minutes, session_log[]` |

> **לוגיקת הניווט** (PRD §5.5): מתחילים ב-`start`; ב-`question` בחירת אופציה
> מנווטת ל-`targetNodeId` (או deep-link); `action` → `nextNodeId`;
> `solution`/`end` מסיימים → משוב. כל מעבר נרשם ל-`session_log[]` (שקוף למשתמש).

## וריאנט — תצוגה ציבורית (SharedGuide)

`SharedGuideLink` (token, public_url, view_count) — תצוגה ציבורית דרך טוקן, ללא
התחברות. משתמשת באותו UI של הנגן, במצב "קריאה בלבד": בלי כותרת אישית, משוב מקוצר
או ללא, ומונה צפיות. נטפל בו כווריאנט קל של הנגן כשנגיע לשיתוף החיצוני.
