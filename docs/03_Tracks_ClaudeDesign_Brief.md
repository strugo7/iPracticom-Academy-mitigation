# בריף ל-Claude Design — מסך ההכשרות (תצוגת משתמש רגיל)

> **מטרה:** לעצב את דף הגישה למסלול ההכשרה של המשתמש, בעקביות עם הדשבורד ועם ה-System Design הקיים.
> **תפקיד הדף (מ-PRD §11):** דף `Learning` — רשימת מסלולי הלמידה.
> **מיקוד נוכחי:** תצוגת **המשתמש הרגיל** (`system_role = user`) שמשויך ל**מסלול אחד בלבד**.
> **הפלט:** עיצוב ויזואלי → יוזן בהמשך ל-Claude Code למימוש מול Supabase.

---

## שם הדף — המלצה

"הכשרות עובדים" → מומלץ לעדכן. השאר את "הכשרה" (תואם לדומיין ולישות `LearningTrack`),
הסר את "עובדים" (מיותר ומוסר קור).

- **תווית ניווט:** "הכשרות"
- **כותרת בדף (משתמש רגיל):** "ההכשרות שלי"
- **כותרת בדף (אדמין/מנהל בעתיד):** "כל ההכשרות"

ניתן להחלפה בקלות — מילה אחת.

---

## עיקרון מפתח — רכיבים לשימוש חוזר

למשתמש הרגיל יש מסלול אחד, אז הוא יראה **כרטיס בודד**. אבל נעצב:
1. `TrackCard` — כרטיס מסלול בודד (רכיב עצמאי).
2. `TracksCatalog` — רשת הקטלוג (layout עצמאי).

נאכלס אותם כעת בכרטיס אחד, אך נבנה אותם כך שיתרחבו לתצוגות המנהל/המדריך/האדמין
(כמה כרטיסים) **ללא עיצוב מחדש**. זה החיסכון הגדול בשלב הזה.

## 📋 הפרומפט — העתק מכאן

```
אני רוצה לעצב את מסך ההכשרות של iPracticom Academy — תצוגת המשתמש הרגיל.
חשוב מאוד: השתמש ברכיבים, בטוקנים ובשפת העיצוב הקיימים בפרויקט (כולל
מה שנבנה במסך הדשבורד) — עקביות מלאה, אל תיצור שפה חדשה.

— זהות מותג (לאכיפה) —
• עברית מלאה, RTL.
• צבעים: ראשי #2EB4FF, כחול כהה #0075DB, אפור #757D86, כהה #181D24,
  רקע בהיר #F4FBFF.
• שפה: tech-human, נקייה, אייקוני outline, אלמנטים גאומטריים, אנימציות עדינות.
• לוגו שמאל-עליון ב-50% שקיפות.
• פוטר: © iPracticom Academy - לשימוש פנימי בלבד

— מהות המסך —
דף "ההכשרות שלי". המשתמש הוא עובד רגיל המשויך למסלול הכשרה אחד.
המטרה: שיראה את המסלול שלו, את ההתקדמות בו, וייכנס בקליק להמשיך.
בנה שני רכיבים לשימוש חוזר: TrackCard (כרטיס מסלול) ו-TracksCatalog
(רשת קטלוג). אכלס כעת בכרטיס אחד, אבל ה-layout צריך לתמוך בכמה כרטיסים
בעתיד (לתפקידים אחרים).

— מבנה הכרטיס (TrackCard) —
• תמונת thumbnail בראש הכרטיס — רלוונטית למחלקה.
• תגית מחלקה/קטגוריה (למשל: טכנאי שטח / מוקד תמיכה / תפעול - Customer Success).
• כותרת המסלול.
• תיאור קצר (שורה-שתיים).
• מטא קטן: רמת קושי + שעות מוערכות.
• בר התקדמות: אחוז השלמה ביחס לכל התוכן במסלול, עם כיתוב "X מתוך Y שיעורים".
• אינדיקציית סטטוס: טרם התחיל / בתהליך / הושלם.
• כפתור פעולה משתנה לפי סטטוס: "התחל ללמוד" / "המשך" / "הושלם ✓".

— כותרת הדף —
"ההכשרות שלי" + שורת תת-כותרת קצרה ("המסלול שהוקצה לך").

— מצבים נדרשים —
• בתהליך (ברירת מחדל לדמו) — כרטיס עם בר התקדמות חלקי, כפתור "המשך".
  השתמש בנתוני דמו בעברית: מסלול "הכשרת טכנאי שטח", 7 מתוך 12 שיעורים, 58%.
• טרם התחיל — בר ריק, כפתור "התחל ללמוד".
• הושלם — טיפול ויזואלי עדין של השלמה (✓), אפשרות "צפה בתעודה".
• ריק (משני) — משתמש בלי מסלול מוקצה: מצב ריק עם איור גאומטרי קטן
  ו"עדיין לא שויכת למסלול הכשרה — פנה למנהל שלך".

— אילוצים טכניים —
React + Tailwind + shadcn/ui, אייקוני lucide-react (outline),
אנימציות עדינות (framer-motion). שמור על רכיבים קיימים — אל תפשט/תמיר.

תתחיל ממצב "בתהליך" בדסקטופ, ואחר כך נדייק.
```

## אנטומיית הכרטיס — רפרנס (שדות SRS)

| אלמנט | שדה נתונים (SRS) | הערה |
|---|---|---|
| Thumbnail | `LearningTrack.image_url` | תמונה לפי מחלקה |
| תגית מחלקה | `LearningTrack.category` | enum: כלל החברה + מחלקות |
| כותרת | `LearningTrack.title` | |
| תיאור | `LearningTrack.description` | |
| רמת קושי | `LearningTrack.difficulty_level` | beginner/intermediate/advanced |
| שעות | `LearningTrack.estimated_hours` | |
| צבע מבטא | `LearningTrack.color` | להתאמת הכרטיס למחלקה |
| התקדמות % | `progress_stats.avg_progress` | ביחס ל-`total_lessons_in_track` |
| X מתוך Y | `lessons_completed` / `total_lessons_in_track` | |
| מסלול מוקצה | `User.assigned_track_id` | למשתמש רגיל — מסלול בודד |

> **חישוב ההתקדמות** (SRS §3.1): המכנה הוא מספר השיעורים ה-`published` במסלול
> (track→TrackModule→SharedModule→Topic→ModuleLesson). הכרטיס רק *מציג* את
> התוצאה מ-`progress_stats` — אין חישוב בצד הלקוח.

## הערה על הרשאות (לשלבים הבאים — לא עכשיו)

הדף הזה משרת ארבע תצוגות מאותם רכיבים:
- **משתמש רגיל** → מסלול אחד (`assigned_track_id`). ← *המיקוד כעת*
- **מנהל** → גישה לכל ההכשרות (לפי הקצאת אדמין).
- **מדריך** → כל ההכשרות, או מסלול ספציפי.
- **אדמין** → הכול.

ההבדל בין התצוגות הוא **אילו מסלולים נשלפים** (סינון בצד ה-backend לפי תפקיד/הקצאה),
לא רכיב אחר. לכן ה-`TracksCatalog` שאנחנו מעצבים עכשיו הוא אותו רכיב לכולם.
