## 1. אנטומיית העורך (Layout — RTL)

```
┌─────────────────────────────────────────────────────────────┐
│  סרגל כלים: [שם Playbook] [קטגוריה][קושי]  [AI] [גרסאות]      │
│             [הרשאות][שיתוף][פרסום]  [שמירה]                   │
├──────────┬──────────────────────────────────┬───────────────┤
│ פלטת      │                                  │  Inspector     │
│ צמתים     │         קנבס (React Flow)          │  (מאפייני      │
│ (6 סוגים) │   רקע נקודות · קשתות bezier ·       │  הצומת הנבחר)  │
│          │   minimap · בקרות זום              │               │
└──────────┴──────────────────────────────────┴───────────────┘
```

- **פלטה** בצד ימין (תחילת הקריאה ב-RTL): 6 כפתורי הצמתים לגרירה/הוספה.
- **קנבס** במרכז: רקע נקודות עדין, קשתות bezier חלקות, minimap (פינה תחתונה),
  בקרות זום/fit/undo-redo.
- **Inspector** בצד שמאל: נפתח כשבוחרים צומת. כאן כל עריכת התוכן.

## 2. הצומת על הקנבס — אנטומיה (per type)

כל צומת = כרטיס עם **פס צבע עליון/מסגרת** לפי הסוג (שפת הצבעים ממסמך 05):
start כחול-כהה · question כחול-מותג · action ענבר · solution ירוק · end אפור ·
linked_flow סגול. כל כרטיס מכיל: אייקון-סוג, כותרת, תצוגה מקדימה קצרה של התיאור,
אינדיקטור מדיה (אם יש), ותגית deep-link (אם מפנה ל-Playbook אחר).

**נקודות חיבור (handles) — הדבר הכי חשוב:**
- `start` — handle יציאה אחד (תחתון).
- `question` — **handle יציאה לכל אופציה** ב-`options[]`, כל אחד מתויג בטקסט
  האופציה. (זה מה שהופך אותו לעץ החלטות — אל תפספס.)
- `action` — handle כניסה + handle יציאה אחד (`nextNodeId`).
- `solution` / `end` — handle כניסה בלבד (צמתי קצה).
- `linked_flow` — נראה כצומת קצה עם תגית "→ Playbook אחר".

**מצבים:** רגיל · נבחר (טבעת הדגשה) · מנותק (אזהרה ויזואלית — אין כניסה/יציאה).

---

## 3. ה-Inspector — עריכת תוכן (per type)

- **כל הסוגים:** כותרת, תיאור, הערה (`note`), מעלה-מדיה (image/gif/video).
- **question:** עורך `options[]` — רשימה ניתנת להוספה/מחיקה/סידור; כל אופציה:
  טקסט, הערה, יעד (`targetNodeId`), מתג deep-link (`linkedFlowId` +
  `linkedTargetNodeId`), קישור חיצוני (`hyperlink`), מדיה.
- **action:** עורך `actions[]` — רשימת פעולות; כל פעולה: טקסט, הערה, deep-link,
  קישור חיצוני, מדיה.
- **linked_flow:** בחירת Playbook יעד + צומת יעד.
- **solution / end:** מינימלי (כותרת + תיאור).

## 📋 פרומפט A — הצמתים והקנבס (העתק מכאן)

```
אני רוצה לעצב את הצמתים והקנבס של עורך ה-Playbooks ב-iPracticom Academy
(מסך FlowEditor). השתמש בשפת העיצוב ובטוקנים הקיימים בפרויקט — עקביות מלאה.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— שפת צבעים לצמתים (לאכיפה) —
start = כחול כהה #0075DB · question = כחול #2EB4FF · action = ענבר #F59E0B ·
solution = ירוק #22C55E · end = אפור #757D86 · linked_flow = סגול #8B5CF6.

— מה לעצב —
1) קנבס בסגנון עורך גרף מקצועי (חשוב על n8n / Figma): רקע נקודות עדין,
   קשתות bezier חלקות, minimap בפינה, בקרות זום/fit.
2) שישה רכיבי צומת מותאמים (custom nodes), כל אחד כרטיס עם: פס צבע עליון לפי
   הסוג, אייקון-סוג outline, כותרת, תצוגה מקדימה קצרה, אינדיקטור מדיה, ותגית
   deep-link אם רלוונטי. מצבים: רגיל / נבחר (טבעת) / מנותק (אזהרה).
3) נקודות חיבור (handles): start = יציאה אחת · question = יציאה לכל אופציה
   (מתויגת בטקסט האופציה!) · action = כניסה+יציאה · solution/end = כניסה בלבד ·
   linked_flow = צומת קצה עם תגית "→ Playbook אחר".

— הצג גרף דמו ריאליסטי (כדי לקבע את המראה) —
start "תקלת מצלמת אבטחה"
 → question "האם נורית ה-LED דולקת?" [כן / לא]
    • לא → action "בדוק חיבור חשמל ו-PoE"
    • כן → question "האם המצלמה מופיעה ב-NVR?" [כן / לא]
         • לא → action "בדוק כתובת IP והגדרות רשת"
         • כן → solution "המצלמה תקינה — בדוק הגדרות תצוגה"
 + צומת end "הסלם לתמיכה טכנית"

— טכני: React + Tailwind + shadcn, lucide-react (outline). הצמתים צריכים להיות
רכיבים עצמאיים שאפשר יהיה לחבר אחר כך ל-@xyflow/react.

תתחיל מהקנבס עם גרף הדמו בדסקטופ.
```

## 📋 פרומפט B — הכרום של העורך (אחרי שהצמתים מוכנים)

```
עכשיו עצב את הכרום שעוטף את הקנבס של עורך ה-Playbooks. אותה שפת עיצוב.

— Layout (RTL): פלטת צמתים בצד ימין · קנבס במרכז · Inspector בצד שמאל ·
  סרגל כלים עליון.

1) סרגל כלים עליון: שם ה-Playbook (לעריכה), בוחרי קטגוריה ורמת קושי,
   כפתורי: "עוזר AI", "היסטוריית גרסאות", "הרשאות עריכה", "שיתוף", מתג "פורסם",
   וכפתור "שמירה" ראשי.

2) פלטת צמתים (ימין): 6 כפתורי הצמתים לגרירה לקנבס, צבע-מקודדים.

3) Inspector (שמאל) — נפתח עם בחירת צומת. הצג שתי גרסאות:
   • צומת question: שדות כותרת/תיאור/הערה/מדיה + עורך options (רשימה
     להוספה/מחיקה/סידור; כל אופציה: טקסט, יעד, מתג deep-link, קישור חיצוני, מדיה).
   • צומת action: כותרת/תיאור/מדיה + עורך actions (רשימת פעולות).

4) פאנל "עוזר AI" (מודאל/צד): כפתור "נתח את ה-Playbook", ופלט מובנה —
   סיכום, בעיות לוגיות (צמתים מנותקים), הצעות שיפור, וגשרים מוצעים עם "החל".

5) פאנל "היסטוריית גרסאות": רשימת גרסאות (מספר, תאריך, שומר, הערת שינוי,
   מספר צמתים/חיבורים) עם "שחזר".

6) מודאל "הרשאות עריכה": בחירת מצב (owner_only / role_based / specific_users)
   + רשימת אימיילים מורשים. מודאל "שיתוף": מתג שיתוף חיצוני + טוקן + מחלקות.

— מובייל: העורך הוא כלי דסקטופ; במובייל הצג הודעה "העריכה זמינה במסך רחב"
  (אבל הנגן כן מותאם מובייל — מסך נפרד).

— טכני: React + Tailwind + shadcn. שמור על רכיבים קיימים.
```

## 📋 פרומפט C — תוספות לעריכה אינטראקטיבית של הצמתים

> הרץ אחרי שפרומפטים A ו-B מוכנים. שכלולים לחוויית העריכה.
> הערה: מדיה-לכל-אופציה כבר נתמכת בסכמה (`options[].media[]` ב-SRS §1.8.1) — מיפוי ישיר.

```
שלוש תוספות לעורך ה-Playbooks. אותה שפת עיצוב, RTL.

1) לחיצה כפולה לעריכה:
   • לחיצה בודדת על צומת = בחירה בלבד (טבעת הדגשה).
   • לחיצה כפולה על צומת = פותחת את עורך הצומת לעריכת כל שדותיו
     (כותרת, תיאור, הערה, מדיה, ו-options/actions לפי הסוג).
     עצב כ-Inspector ממוקד או drawer/מודאל עם מספיק מרחב לעריכה עשירה.

2) מדיה לכל אופציה:
   • בעורך ה-options של צומת שאלה — לכל אופציה חריץ מדיה משלה:
     העלאת תמונה או GIF, תצוגה מקדימה ממוזערת (thumbnail), והסרה/החלפה.
   • הצג אופציה אחת עם תמונה מצורפת ואחת בלי — לראות את שני המצבים.

3) הזזת צמתים חופשית:
   • הצמתים נגררים חופשית בכל רחבי הקנבס.
   • עצב את מצב הגרירה: הצומת "מתרומם" עם צל עדין, הסמן grab → grabbing,
     וקו-עזר עדין ליישור (אופציונלי).

טכני: React + Tailwind + shadcn. הגרירה והלחיצה הכפולה יחוברו ל-@xyflow/react
(draggable nodes, onNodeDoubleClick) — עצב את המראה והמצבים.
```

## אנטומיה — רפרנס שדות (SRS §1.8.1)

| אלמנט | שדה |
|---|---|
| צומת | `nodes[].{id, type, title, description, note, position, media[], nextNodeId}` |
| אופציה (question) | `options[].{id, text, note, targetNodeId, linkedFlowId, linkedTargetNodeId, hyperlink, media}` |
| פעולה (action) | `actions[].{text, note, linkedFlowId, linkedTargetNodeId, hyperlink, media}` |
| חיבור | `connections[].{id, sourceNodeId, targetNodeId, optionIndex, label}` |
| מדיה | `media[].{id, type(image/gif/video), url, alt, size}` |
| גרסאות | `version`, `version_history[].{version, saved_at, saved_by_name, change_note, node_count, connection_count, snapshot}` |
| הרשאות | `edit_permissions.{mode, allowed_emails[]}` |
| שיתוף | `share_settings.{is_shared_externally, share_token, shared_with_departments[]}` |
| עוזר AI | `troubleshooting_flow_builder` → `{summary, improvement_suggestions, logic_issues, proposed_changes}` |

