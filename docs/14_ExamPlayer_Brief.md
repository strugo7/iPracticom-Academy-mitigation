# בריף עיצוב — מערכת ביצוע המבחן (Exam Player)

> **מטרה:** עיצוב חוויית ביצוע המבחן — שאלות, טיימר, ניווט, וסקירה לפני הגשה.
> **תפקיד (PRD §11):** `QuizPage` / `ExamPage` — כולם.
> **מקורות:** SRS §1.4 (Exam: `time_limit`, `questions[]`), ExamAttempt (`current_index`, `user_answers`, `status`, `detailed_results`).
> **מקביל ל:** FlowPlayer (הרצת Playbook), אך להערכה.

---

## 1. שני מסכים

1. **מסך ביצוע** — שאלה נוכחית + טיימר + ניווט-grid.
2. **מסך Overview (סקירה)** — כל השאלות והתשובות שסומנו → הגשה סופית.

> אחרי ההגשה הסופית: ציון + עבר/נכשל. הפירוט המלא לכל שאלה (תשובה נכונה +
> הסבר) חי ב-drill-down של עמוד הפרופיל (מסמך 09).

---

## 2. ניווט-Grid — האלמנט המרכזי

רשת ריבועים ממוספרים (1, 2, 3…). לחיצה על ריבוע → קפיצה לשאלה. מצבים:

| מצב | מראה |
|---|---|
| נענתה | ירוק |
| מסומנת בדגל | כתום + אייקון דגל |
| דולגה (בוקרה, לא נענתה) | אדום |
| נוכחית | טבעת הדגשה |
| טרם נצפתה | נייטרלי/בהיר |

+ מקרא צבעים. ב-RTL: פאנל הניווט בצד, השאלה במרכז.

---

## 3. רינדור 4 סוגי השאלות (למענה)

- `multiple_choice` — אפשרויות כרדיו גדולים.
- `true_false` — שני כפתורים.
- `matching` — גרירת התאמות (ימני ↔ שמאלי).
- `order_sequence` — גרירת פריטים לסדר.

לכל שאלה: כפתור **דגל** (סימון לחזרה).

---

## 4. טיימר

ספירה לאחור מ-`time_limit` (ברירת מחדל 20 דק'). בולט בכותרת. מתחת ל-5 דק' → ענבר;
מתחת לדקה → אדום. ב-0 → **הגשה אוטומטית**.

---

## 5. הגשה דו-שלבית

1. כפתור "הגש מבחן" ליד השעון → אם יש שאלות לא-עניות: אזהרה "יש N שאלות שלא נענו".
2. **מסך Overview:** רשימת כל השאלות + התשובה שסומנה לכל אחת + מצב דגל; קפיצה
   לעריכה לכל שאלה.
3. **הגשה סופית** אחרי הסקירה.

---

## 📋 הפרומפט — העתק מכאן

```
אני רוצה לעצב את מערכת ביצוע המבחן (Exam Player) של iPracticom Academy.
השתמש בשפת העיצוב והטוקנים הקיימים — עקביות מלאה, RTL. חוויה ממוקדת ורגועה
(ביצוע מבחן מלחיץ — אל תוסיף עומס מיותר).

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— מסך הביצוע —
• כותרת עליונה: שם המבחן · טיימר ספירה-לאחור בולט (20:00) · כפתור "הגש מבחן".
  הטיימר הופך לענבר מתחת ל-5 דק' ולאדום מתחת לדקה.
• אזור ראשי: השאלה הנוכחית — מספר + נוסח + מדיה אם יש + אזור מענה לפי סוג
  (רב-ברירה=רדיו גדולים / נכון-לא נכון=2 כפתורים / התאמה=גרירה / סידור=גרירה).
  כפתור "סמן בדגל" על השאלה.
• ניווט תחתון: "הקודמת" / "הבאה".
• פאנל ניווט (צד): grid של ריבועים ממוספרים. צבעים: נענתה=ירוק,
  מסומנת בדגל=כתום+אייקון דגל, דולגה=אדום, נוכחית=טבעת הדגשה,
  טרם נצפתה=נייטרלי. לחיצה על ריבוע → קפיצה לשאלה. כולל מקרא.

— מסך Overview (בלחיצה על "הגש מבחן") —
• אם יש שאלות לא-עניות: אזהרה "יש לך N שאלות שלא נענו" + קישור אליהן.
• רשימת כל השאלות עם התשובה שסומנה לכל אחת + מצב דגל. קפיצה לעריכה לכל שאלה.
• כפתור "הגשה סופית" (לאחר סקירה).

— נתוני דמו: מבחן "יסודות רשתות", 10 שאלות, 20 דק'. שאלה נוכחית 4 (רב-ברירה
  על MikroTik). ב-grid: 1-3 ירוק, 5 כתום (דגל), 6-7 אדום (דולגו), 4 נוכחית,
  8-10 נייטרלי.

— מצבים: דסקטופ-first + מובייל (ה-grid כ-drawer/שורה נגללת).
— טכני: React + Tailwind + shadcn/ui, lucide-react (outline), dnd-kit
  (התאמה/סידור), framer-motion. שמור על רכיבים קיימים.

תתחיל ממסך הביצוע בדסקטופ עם השאלה הנוכחית וה-grid מאוכלס, ואז Overview.
```

## אנטומיה — רפרנס שדות

| אלמנט | שדה |
|---|---|
| טיימר | `Exam.time_limit` (דקות) |
| שאלות | `Exam.questions[]` → `Question.{question_text, question_type, options, order_items, media}` |
| מענה משתמש | `ExamAttempt.user_answers` |
| מיקום נוכחי | `ExamAttempt.current_index` |
| מצב | `ExamAttempt.status` (in_progress/submitted) |
| תוצאה | `ExamAttempt.{score, passed}` + `detailed_results` (לפרופיל) |

> **ערבוב (memory):** סדר השאלות/אפשרויות עשוי להיות מעורבב לפי `seed` עקבי לכל
> ניסיון — לשמר את ה-seed לשחזור זהה.

  (במקום כתום מלא), כדי לראות את שני המצבים."

- "הראה את מצב הטיימר מתחת לדקה (אדום + פעימה עדינה)."
- "הראה את מסך ה-Overview עם 10 שאלות."
- "הראה גרסת מובייל."
