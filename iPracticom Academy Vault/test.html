שכבה 7 - Application Layer

שכבת היישום - הממשק עם המשתמש

מה זה?

שכבת היישום היא השכבה הקרובה ביותר למשתמש הקצה. היא מספקת את הממשק שבאמצעותו יישומים מתקשרים עם הרשת. שכבה זו אחראית על פרוטוקולים שמאפשרים תקשורת בין תוכנות.

פרוטוקולים ושירותים

  • HTTP/HTTPS - גלישה באינטרנט (פורט 80/443)
  • FTP - העברת קבצים (פורט 20/21)
  • SMTP - שליחת דואר (פורט 25/587)
  • POP3/IMAP - קבלת דואר (פורט 110/143)
  • DNS - תרגום שמות (פורט 53)
  • SSH - גישה מאובטחת מרחוק (פורט 22)
  • Telnet - גישה מרחוק (פורט 23)
  • DHCP - הקצאת כתובות IP (פורט 67/68)

התקנים ותוכנות

  • דפדפנים (Chrome, Firefox, Edge)
  • תוכנות דואר (Outlook, Gmail)
  • תוכנות FTP (FileZilla)
  • שרתי Web (Apache, Nginx, IIS)
  • שרתי DNS
  • Firewalls ברמת Application (WAF)

תקלות נפוצות

  • DNS Resolution Failed - בעיה בתרגום שם לכתובת IP
  • HTTP 404/500 - שגיאות שרת או דף לא נמצא
  • SSL Certificate Error - בעיה באישור אבטחה
  • Authentication Failed - כשל באימות משתמש
  • Timeout - זמן התגובה חרג
  • CORS Errors - בעיות הרשאות Cross-Origin

כלי אבחון

  • nslookup - בדיקת DNS
  • curl / wget - בדיקת HTTP
  • Developer Tools בדפדפן
  • telnet [host] [port] - בדיקת חיבור

שכבה 4 - Transport Layer

שכבת התעבורה - אמינות וזרימת נתונים

מה זה?

שכבת התעבורה אחראית על העברת נתונים מקצה לקצה (End-to-End). היא מבטיחה שהנתונים יגיעו בשלמותם, בסדר הנכון, ומנהלת בקרת זרימה ותיקון שגיאות.

TCP vs UDP

  • TCP (Transmission Control Protocol)
    • מבוסס חיבור (Connection-oriented)
    • אמין - מבטיח קבלת כל הנתונים
    • Three-Way Handshake (SYN, SYN-ACK, ACK)
    • שימושים: HTTP, FTP, Email
  • UDP (User Datagram Protocol)
    • ללא חיבור (Connectionless)
    • מהיר יותר, פחות אמין
    • שימושים: DNS, VoIP, Streaming, Gaming

מושגי מפתח

  • Segments - יחידת הנתונים בשכבה זו
  • Port Numbers - מזהים של יישומים (0-65535)
  • Flow Control - בקרת קצב העברה
  • Windowing - כמות נתונים לשליחה לפני ACK
  • Sequence Numbers - סדר הסגמנטים

תקלות נפוצות

  • Port Already in Use - פורט תפוס ע"י תהליך אחר
  • Connection Refused - השרת לא מקשיב על הפורט
  • Connection Timeout - לא נוצר חיבור בזמן
  • RST Packets - חיבור נקטע באופן פתאומי
  • Retransmissions - שליחות חוזרות (עומס/אובדן)
  • Port Exhaustion - מיצוי פורטים זמינים

כלי אבחון

  • netstat -an - צפייה בחיבורים פעילים
  • ss -tuln - חיבורים (Linux)
  • tcpdump - לכידת מנות
  • Wireshark - ניתוח תעבורה מתקדם
  • telnet [ip] [port] - בדיקת פורט פתוח

סוגי פורטים

  • Well-Known (0-1023) - פורטים שמורים לשירותים סטנדרטיים
  • Registered (1024-49151) - רשומים ל-IANA
  • Dynamic/Private (49152-65535) - פורטים זמניים

שכבה 3 - Network Layer

שכבת הרשת - ניתוב וכתובות לוגיות

מה זה?

שכבת הרשת אחראית על ניתוב (Routing) של מנות נתונים בין רשתות שונות. היא משתמשת בכתובות לוגיות (IP Addresses) כדי לקבוע את הנתיב הטוב ביותר להעברת הנתונים.

פרוטוקולים עיקריים

  • IP (Internet Protocol) - פרוטוקול הכתובות הבסיסי
  • ICMP - בקרה והודעות שגיאה (Ping)
  • ARP - תרגום IP ל-MAC
  • OSPF - פרוטוקול ניתוב פנימי
  • BGP - ניתוב בין ארגונים
  • RIP - פרוטוקול ניתוב פשוט

התקנים

  • Router (נתב) - מנתב תעבורה בין רשתות
  • Layer 3 Switch - מתג עם יכולות ניתוב
  • Firewall - סינון תעבורה לפי IP
  • Load Balancer - איזון עומסים

כתובות IP

  • IPv4 - 32 ביט (192.168.1.1)
  • IPv6 - 128 ביט (2001:db8::1)
  • Subnet Mask - הפרדה בין רשת למחשב
  • Default Gateway - הנתב ליציאה מהרשת
  • Private IPs - 10.x.x.x, 172.16-31.x.x, 192.168.x.x

תקלות נפוצות

  • Destination Unreachable - אין נתיב ליעד
  • TTL Expired - המנה "מתה" בדרך
  • IP Conflict - שני מכשירים עם אותו IP
  • Wrong Subnet - הגדרת רשת שגויה
  • No Default Gateway - אין יציאה מהרשת המקומית
  • Routing Loop - מנות סובבות במעגל
  • MTU Issues - גודל מנה לא מתאים

כלי אבחון

  • ping [ip/host] - בדיקת קישוריות
  • tracert / traceroute - מעקב נתיב
  • ipconfig / ifconfig - הגדרות IP
  • route print - טבלת ניתוב
  • arp -a - טבלת ARP
  • pathping - שילוב ping ו-tracert

שכבה 2 - Data Link Layer

שכבת קישור הנתונים - תקשורת מקומית

מה זה?

שכבת קישור הנתונים אחראית על העברת נתונים בין שני התקנים באותה רשת מקומית (LAN). היא משתמשת בכתובות פיזיות (MAC Addresses) ומטפלת בזיהוי ותיקון שגיאות.

תתי-שכבות

  • LLC (Logical Link Control)
    • בקרת זרימה
    • זיהוי שגיאות
    • ממשק לשכבה 3
  • MAC (Media Access Control)
    • כתובות פיזיות
    • גישה למדיה (CSMA/CD)
    • מסגור (Framing)

התקנים

  • Switch (מתג) - מחבר התקנים ברשת מקומית
  • Bridge - מחבר סגמנטים של רשת
  • NIC (Network Interface Card) - כרטיס רשת
  • Access Point - נקודת גישה אלחוטית

כתובת MAC

  • 48 ביט (6 בתים)
  • פורמט: AA:BB:CC:DD:EE:FF
  • 3 בתים ראשונים - יצרן (OUI)
  • 3 בתים אחרונים - מזהה ייחודי
  • כתובת שידור: FF:FF:FF:FF:FF:FF

פרוטוקולים וטכנולוגיות

  • Ethernet (802.3) - רשת קווית
  • Wi-Fi (802.11) - רשת אלחוטית
  • STP (Spanning Tree) - מניעת לולאות
  • VLAN - רשתות וירטואליות
  • PPP - Point-to-Point Protocol

תקלות נפוצות

  • Duplicate MAC - כתובות MAC כפולות
  • Switching Loops - לולאות ברשת (STP Issue)
  • VLAN Mismatch - אי התאמת VLANs
  • Broadcast Storm - הצפת שידורים
  • Port Security Violation - הפרת אבטחת פורט
  • Duplex Mismatch - אי התאמת Full/Half
  • Native VLAN Mismatch - בעיית Trunk

כלי אבחון

  • arp -a - טבלת MAC-IP
  • getmac - כתובת MAC מקומית
  • show mac address-table - בSwitch
  • show spanning-tree - מצב STP
  • Wireshark - ניתוח מסגרות

שכבה 1 - Physical Layer

השכבה הפיזית - חומרה ואותות

מה זה?

השכבה הפיזית היא הבסיס של מודל OSI. היא עוסקת בהעברת ביטים גולמיים דרך מדיה פיזית - כבלים, סיבים אופטיים או גלי רדיו. שכבה זו מגדירה מאפיינים חשמליים, מכניים ופונקציונליים.

סוגי כבלים

  • UTP (Unshielded Twisted Pair)
    • Cat5e - עד 1Gbps
    • Cat6 - עד 10Gbps (קצר)
    • Cat6a - עד 10Gbps (100m)
  • STP (Shielded) - מוגן מהפרעות
  • Fiber Optic
    • Single-mode - מרחקים ארוכים
    • Multi-mode - מרחקים קצרים
  • Coaxial - כבל קואקסיאלי

התקנים

  • Hub - מרכז שמשדר לכל הפורטים
  • Repeater - מגבר אותות
  • Media Converter - המרת סוגי מדיה
  • Modem - מודולציה/דמודולציה
  • Patch Panel - לוח חיבורים
  • Network Cables - כבלי רשת

מדיה אלחוטית

  • Wi-Fi - 2.4GHz / 5GHz / 6GHz
  • Bluetooth - טווח קצר
  • Infrared - אינפרא-אדום
  • Cellular - 4G/5G
  • Satellite - תקשורת לווין

מושגים חשובים

  • Bandwidth - רוחב פס (Mbps/Gbps)
  • Throughput - תפוקה בפועל
  • Latency - השהייה
  • Attenuation - דעיכת אות
  • Crosstalk - הפרעות בין כבלים
  • EMI - הפרעות אלקטרומגנטיות

תקלות נפוצות

  • כבל פגום - קרע, קיפול, לחיצה
  • חיבור רופף - RJ45 לא מחובר היטב
  • Attenuation - כבל ארוך מדי
  • EMI/RFI - הפרעות חשמליות
  • Crosstalk - הפרעות בין זוגות
  • אות Wi-Fi חלש - מרחק/מכשולים
  • Port מת - פורט פיזי לא עובד
  • LED לא דולק - אין חיבור פיזי

כלי אבחון

  • Cable Tester - בודק כבלים
  • Tone Generator - מעקב כבלים
  • Multimeter - מודד מתח
  • OTDR - בדיקת סיב אופטי
  • Spectrum Analyzer - ניתוח אלחוטי
  • בדיקת LEDs על ציוד

אנימציית Encapsulation

צפה כיצד נתונים עוטפים בכותרות בכל שכבה בדרך למטה (Encapsulation) ונפתחים בדרך למעלה (Decapsulation)
