# בריף עיצוב — אשכול ההגדרות (Settings)

> **מטרה:** עיצוב שלושת מסכי ההגדרות, נגישים מ-dropdown בסרגל הניווט.
> **תפקיד (PRD §11):** הגדרות מערכת/אבטחה/משתמשים — admin (חלק manager+).
> **מקורות:** SRS — AppSetting, SecurityLog, Department, User, Invite, generateInviteToken.

---

## 1. ה-dropdown בסרגל הניווט

פריט "הגדרות" (בתחתית הסרגל, ממסמך 11) נפתח לתת-תפריט:
1. **הגדרות מערכת** (System Settings) — admin
2. **לוגי אבטחה** (Security Logs) — admin/manager
3. **ניהול משתמשים ומבנה ארגוני** — admin

> עדכון קל למעטפת (מסמך 11): פריט "הגדרות" → submenu נפתח.

---

## 2. הישויות (מחקר)

| מסך | ישות | הערה |
|---|---|---|
| הגדרות מערכת | `AppSetting` (key/value JSON) | להציג כטפסים מקובצים, לא JSON |
| לוגי אבטחה | `SecurityLog` | viewer לקריאה, RLS admin/manager |
| מבנה ארגוני | `Department` (**היררכי — parent_id**) | עץ, לא רשימה |
| משתמשים | `User` | תפקיד, סטטוס, שיוך מחלקה |
| הזמנות | `Invite` + `generateInviteToken` | טוקן hashed (random+SHA256) |

## 📋 פרומפט A — הגדרות מערכת (העתק מכאן)

```
אני רוצה לעצב את עמוד הגדרות המערכת של iPracticom Academy (לאדמין).
השתמש בשפת העיצוב והטוקנים הקיימים — עקביות מלאה, RTL.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— מבנה: עמוד הגדרות עם ניווט-סקשנים (טאבים צדדיים או כרטיסים). הצג כטפסים
  ידידותיים (לא JSON). סקשנים:
  • מיתוג: לוגו, צבעי מותג, טקסט פוטר.
  • אבטחת התחברות: רשימת דומיינים מותרים (whitelist) להוספה/הסרה, הגבלות IP.
  • ברירות מחדל: זמן מבחן ברירת מחדל, XP לשיעור, ציון מעבר.
  • אינטגרציות: סטטוס מפתחות (OpenRouter, n8n, R2, Gamma, Pexels) — ממוסך,
    עם אינדיקטור "מחובר/לא מחובר".
  • מסמכים/PDF: הגדרות הפקת PDF.
  • כפתור "סריקת אבטחה" (runSecurityAudit).
• כל סקשן עם כפתור "שמור" משלו.

— מצבים: דסקטופ-first + מובייל. הצג את סקשן "אבטחת התחברות".
— טכני: React + Tailwind + shadcn/ui, lucide-react (outline). שמור על רכיבים קיימים.

תתחיל מסקשן אבטחת ההתחברות בדסקטופ.
```

## 📋 פרומפט B — לוגי אבטחה (העתק מכאן)

```
עכשיו עצב את עמוד לוגי האבטחה. אותה שפת עיצוב, RTL. עמוד לקריאה בלבד.

— מבנה —
• כרטיסי סיכום עליונים: ניסיונות חסומים היום, כשלי 2FA, חסימות whitelist,
  חריגות rate-limit.
• שורת פילטרים: סוג ניסיון (התחברות-דומיין-לא-מורשה / כשל-2FA / חסימת-whitelist /
  התחברות / חריגת-rate-limit / אחר), סטטוס (חסום/הצלחה/שגיאה), טווח תאריכים,
  חיפוש לפי אימייל/IP.
• טבלת לוגים: חותמת זמן, אימייל, סוג ניסיון (תווית ידידותית), סטטוס
  (חסום=אדום, הצלחה=ירוק, שגיאה=ענבר), IP, נתיב. הרחבת שורה → metadata מלא.
• כפתור "ייצוא".

— נתוני דמו: 6 רשומות מגוונות בעברית (חסימת whitelist, כשל 2FA, התחברות תקינה,
  חריגת rate-limit).
— מצבים: דסקטופ + מובייל. טכני: React + Tailwind + shadcn, recharts לסיכום
  אם צריך. שמור על רכיבים קיימים.

תתחיל מהטבלה המאוכלסת עם כרטיסי הסיכום בדסקטופ.
```

## 📋 פרומפט C — מבנה ארגוני, משתמשים והזמנות (העתק מכאן)

```
עכשיו עצב את עמוד ניהול המשתמשים והמבנה הארגוני. אותה שפת עיצוב, RTL.

— פריסה: שני חלונות —
• עץ מחלקות (צד ימין): היררכיה (מחלקות ותת-מחלקות, parent_id), כל מחלקה עם
  מונה חברים. פעולות: הוסף מחלקה/תת-מחלקה, ערוך, סדר מחדש (drag).
• משתמשי המחלקה הנבחרת (צד שמאל): טבלה — שם+אווטאר, אימייל, תפקיד
  (משתמש/מדריך/מנהל/אדמין), סטטוס (פעיל/לא-פעיל), פעילות אחרונה.
  פעולות: ערוך תפקיד, העבר מחלקה, השבת/הפעל.
• כפתורים: "הזמן משתמש" (למחלקה הנבחרת) + "מחלקה חדשה".

— זרימת הזמנה (מודאל) —
• טופס: אימייל, שם מלא, מחלקה (מאוכלסת מראש מהנבחרת), תפקיד.
• יצירה → מציג magic link להעתקה/שליחה.
• לשונית/סקשן "הזמנות ממתינות": רשימה עם סטטוס (ממתינה/פגה/הושלמה),
  מזמין, תאריך תפוגה, ופעולות "שלח שוב" / "בטל".

— נתוני דמו (עברית) —
עץ: הנהלה → (תפעול, תמיכה טכנית → טכנאי שטח · מוקד תמיכה, מכירות).
משתמשים במחלקת "טכנאי שטח": 3-4 עם תפקידים שונים. 2 הזמנות ממתינות.

— מצבים: דסקטופ-first + מובייל (העץ כ-drawer). הצג עם מחלקה נבחרת ומשתמשיה,
  ואת מודאל ההזמנה.
— טכני: React + Tailwind + shadcn/ui, dnd-kit (עץ), lucide-react. שמור על רכיבים.

תתחיל מהמבט המלא: עץ מחלקות + טבלת משתמשים של מחלקה נבחרת.
```

## אנטומיה — רפרנס שדות

| אזור | שדות |
|---|---|
| הגדרות | `AppSetting.{key, value(JSON), description}` |
| לוגים | `SecurityLog.{email, ip_address, user_agent, attempt_type, status, path, details, metadata}` |
| מחלקה | `Department.{name, parent_id, order_index, description}` |
| משתמש | `User.{full_name, email, system_role, department, status, last_activity}` |
| הזמנה | `Invite.{invited_by_user_id, ...}` + `generateInviteToken{email, department, full_name}` → `{invite_id, magic_link, expires_at}` |
