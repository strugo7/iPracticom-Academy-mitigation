# בריף ל-Claude Design — מסך הבית (Dashboard אישי)

> **מטרה:** להנחות את Claude Design לעצב את הדשבורד האישי של המשתמש, בעקביות עם ה-System Design הקיים.
> **תפקיד הדף (מ-PRD §11):** דשבורד אישי, ברירת המחדל לכל המשתמשים.
> **הפלט:** עיצוב ויזואלי → יוזן בהמשך ל-Claude Code למימוש מול Supabase.

---

## איך לעבוד עם המסמך הזה

1. פתח את הפרויקט הקיים שלך ב-Claude Design (אותו קנבס של ה-System Design).
2. העתק את **הפרומפט** למטה והדבק בצ'אט.
3. אחרי שהמסך נוצר — דייק עם **פרומפטי האיטרציה**.
4. סיים עם **בקשת ה-handoff** כדי שהמסך יהיה מוכן למסירה ל-Claude Code.

---

## 📋 הפרומפט — העתק מכאן

```
אני רוצה לעצב את מסך הבית (Dashboard אישי) של iPracticom Academy.
חשוב מאוד: השתמש ברכיבים, בטוקנים ובשפת העיצוב שכבר קיימים במערכת העיצוב
של הפרויקט הזה — אל תיצור שפה חדשה. עקביות מלאה עם המסכים הקיימים.

— זהות מותג (לאכיפה) —
• עברית מלאה, RTL.
• צבעים: ראשי #2EB4FF, כחול כהה #0075DB, אפור #757D86, כהה #181D24,
  רקע בהיר #F4FBFF.
• שפה: tech-human, נקייה, אייקוני outline, אלמנטים גאומטריים, אנימציות עדינות.
• לוגו בפינה השמאלית-עליונה ב-50% שקיפות.
• פוטר בתחתית כל מסך: © iPracticom Academy - לשימוש פנימי בלבד

— מהות המסך —
דשבורד אישי של לומד. המטרה: שהמשתמש יראה במבט אחד איפה הוא עומד,
ויחזור בקליק ללמידה. קהל: עובד/לומד (system_role = user). מנהלים רואים
דשבורד נפרד, אז כאן מתמקדים בחוויית הלומד.

— ארכיטקטורת מידע (לפי אזורים) —

1) כותרת עליונה:
   • ברכה אישית: "שלום, [שם]" + תגית תפקיד קטנה.
   • פעמון התראות (מונה).
   • לוגו שמאל-עליון ב-50% שקיפות.

2) אזור "המשך ללמוד" (Hero):
   • כרטיס בולט עם המסלול המוקצה (assigned_track) ושמו.
   • טבעת/בר התקדמות גדול: התקדמות כללית (avg_progress, באחוזים).
   • כיתוב "נשארו X שיעורים".
   • כפתור ראשי: "המשך ללמוד" → השיעור הבא.

3) שורת מדדים (כרטיסי KPI, אייקון outline לכל אחד):
   • נקודות XP (total_xp) + רמה.
   • שיעורים שהושלמו (lessons_completed / total_lessons_in_track).
   • מבחנים שעברת (exams_passed / total_exams).
   • תעודות (certificates_earned).
   • זמן למידה (total_time_spent_minutes → המרה לשעות).

4) פעילות שבועית:
   • שיעורים השבוע (weekly_lessons) + זמן השבוע (weekly_time_spent_minutes).
   • ציון ממוצע (avg_score) — תצוגה ויזואלית עדינה (radial/דונאט).

5) טבלת מובילים (Leaderboard):
   • Top 5 לומדים לפי XP/התקדמות, עם הדגשת המשתמש הנוכחי.

6) פעילות אחרונה (Activity feed):
   • רשימת אירועים אחרונים: "השלמת שיעור X", "קיבלת תעודה", "עברת מבחן".

7) פוטר מותג.

— מצבים נדרשים —
• מצב מאוכלס (ברירת מחדל) — עם נתוני דמו ריאליסטיים בעברית.
• מצב ריק — משתמש חדש בלי מסלול מוקצה: הצג קריאה לפעולה לקליטה/בחירת מסלול
  במקום כרטיסים ריקים.
• מובייל — פריסה רספונסיבית; הכרטיסים נערמים, ה-Hero נשאר בראש.

— אילוצים טכניים —
שמור על מימוש ב-React + Tailwind + shadcn/ui. השתמש ב-lucide-react לאייקונים
(outline). אנימציות עדינות בלבד (framer-motion). אל תפשט או תמיר רכיבים קיימים.

תתחיל מהמצב המאוכלס בדסקטופ, ואחר כך נדייק.
```

## מפרט התוכן — רפרנס מלא (לעיון/התאמה)

| אזור | רכיב | שדה נתונים (SRS) | תווית עברית |
|---|---|---|---|
| Hero | מסלול מוקצה | `assigned_track_id` → `LearningTrack.title` | המסלול שלי |
| Hero | התקדמות | `progress_stats.avg_progress` | התקדמות כללית |
| KPI | XP | `progress_stats.total_xp` | נקודות XP |
| KPI | שיעורים | `lessons_completed` / `total_lessons_in_track` | שיעורים שהושלמו |
| KPI | מבחנים | `exams_passed` / `total_exams` | מבחנים שעברת |
| KPI | תעודות | `certificates_earned` | תעודות |
| KPI | זמן | `total_time_spent_minutes` | זמן למידה |
| שבועי | שיעורים השבוע | `weekly_lessons` | פעילות השבוע |
| שבועי | ציון | `avg_score` | ציון ממוצע |
| מובילים | דירוג | `getLeaderboard()` | טבלת מובילים |
| פיד | אירועים | `getActivityFeed()` | פעילות אחרונה |

> כל המדדים שבכרטיסים מגיעים מאובייקט `User.progress_stats` שמחושב אוטומטית
> ע"י `recalculateUserStats` (SRS §3.1). הדשבורד הוא בעיקר *קריאה* מאובייקט אחד —
> מה שהופך אותו לדף מצוין להתחיל ממנו: פשוט לחיווט, אבל מציג את כל המערכת.

- "טבלת המובילים: הדגש את השורה של המשתמש הנוכחי עם רקע בגוון #2EB4FF עדין."
- "הוסף מצב ריק לפיד הפעילות: איור גאומטרי קטן + 'עדיין אין פעילות — בוא נתחיל'."
- "הפוך את טבעת ההתקדמות ב-Hero לאנימציה עדינה שמתמלאת בטעינה."
- "הראה לי את גרסת המובייל של המסך הזה."
- "צמצם את שורת המדדים ל-3 כרטיסים עיקריים והעבר את השאר ללשונית 'הסטטיסטיקות שלי'." (אם צפוף)

## בקשת ה-handoff (לסיום — לקראת Claude Code)

```
מצוין. עכשיו תכין לי סיכום מימוש למסך הזה שאוכל למסור למפתח:
• רשימת הרכיבים (components) עם שמות.
• מבנה הנתונים (props) שכל רכיב מצפה לקבל — בשמות השדות שציינתי.
• טוקני העיצוב שבהם השתמשת (צבעים, מרווחים, טיפוגרפיה).
• הערות לגבי מצב ריק ומובייל.
שמור על React + Tailwind + shadcn.
```

הפלט הזה הוא בדיוק מה שניתן ל-Claude Code בשלב הבא כדי לממש את הדשבורד מול Supabase.
