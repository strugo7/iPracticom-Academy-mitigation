# בריף ל-Claude Code — שילוב טופולוגיית הרשת כבלוק שיעור

> **מטרה:** לשלב את `network_canvas` שנבנה כבלוק במשפחת הבלוקים האינטראקטיביים
> של עורך השיעורים, ולרנדר אותו **פרוס היטב בתוך השיעור** (תצוגת לומד).
> **קהל:** Claude Code (אינטגרציה + render mode, הבלוק כבר קיים).

---

## 1. הקשר

`network_canvas` נבנה (כולל VLANs). עכשיו: (א) לרשום אותו כבלוק שיעור במשפחה
האינטראקטיבית, (ב) לתת לו **תצוגת לומד מלוטשת** בתוך השיעור הנגלל.

---

## 2. חלק א — אינטגרציה במשפחת הבלוקים

- רשום `network_canvas` ב-block registry / `nodeTypes`-style mapping של העורך
  (מסמך 19, חלק 1).
- הוסף לפלטת הבלוקים תחת **אינטראקטיבי**, עם אייקון ייעודי (רשת/טופולוגיה).
- עטוף ב-`BlockWrapper` הקיים (affordances, סידור, סרגל הקשרי).
- **מצב עריכה בתוך השיעור:** הבלוק מציג תצוגה מקדימה של הטופולוגיה + כפתור
  "ערוך טופולוגיה" שפותח את העורך המלא (drawer/מודאל). מבנה ה-`data`
  (nodes/connections/vlans) זהה.

---

## 3. חלק ב — תצוגת הלומד (הדגש: "פרוס היטב")

הטופולוגיה צריכה להשתלב בשיעור נגלל בלי להשתלט:

- **גובה תחום:** מסגרת/כרטיס בגובה סביר (≈420-520px), לא כל גובה המסך.
- **Fit-to-view:** בטעינה — zoom אוטומטי שמראה את כל הטופולוגיה ממורכזת.
- **אינטראקטיבי לקריאה בלבד:** pan + zoom + לחיצה על רכיב → הגדרותיו/פאנל
  הסוויץ'. **בלי עריכה** (אין גרירה/הוספה/מחיקה).
- **כפתור "מסך מלא":** לטופולוגיות מורכבות — הרחבה ל-overlay מלא.
- **מקרא (legend):** צבעי ה-VLAN וסוגי הרכיבים, מתקפל.
- **כותרת/כיתוב:** מעל הבלוק.
- **מסגרת מותגית:** כרטיס נקי שמשתלב בשיעור.

---

## 4. רספונסיביות

- **מובייל:** fit-to-view + pinch-zoom; אם צפוף — אפשר scroll-to-pan,
  או הצגת כפתור "פתח טופולוגיה במסך מלא" כברירת מחדל.
- שמור קריאוּת — אל תכווץ אייקונים/תוויות מתחת לסף קריא.

---

## 📋 פרומפט ל-Claude Code — העתק מכאן

```
שלב את בלוק network_canvas הקיים כבלוק שיעור בעורך, עם תצוגת לומד מלוטשת.

חלק א — אינטגרציה:
• רשום network_canvas ב-block registry של העורך (כמו שאר הבלוקים, מסמך חלק 1).
• הוסף לפלטת הבלוקים תחת משפחת "אינטראקטיבי", אייקון רשת/טופולוגיה.
• עטוף ב-BlockWrapper הקיים (affordances/סידור/סרגל הקשרי).
• מצב עריכה בשיעור: תצוגה מקדימה + כפתור "ערוך טופולוגיה" שפותח את העורך המלא
  הקיים (drawer/מודאל). אותו data (nodes/connections/vlans).

חלק ב — תצוגת לומד (read-only) פרוסה היטב:
• מסגרת/כרטיס בגובה תחום (≈420-520px), עם כותרת/כיתוב מעל.
• fit-to-view אוטומטי בטעינה — כל הטופולוגיה ממורכזת ונראית.
• אינטראקציה לקריאה בלבד: pan + zoom + לחיצה על רכיב → הגדרות/פאנל סוויץ'.
  בלי גרירה/עריכה.
• כפתור "מסך מלא" (overlay) לטופולוגיות מורכבות.
• מקרא מתקפל: צבעי VLAN + סוגי רכיבים.
• רספונסיבי: מובייל = fit + pinch-zoom, ואם צפוף — "פתח במסך מלא".

שמור על RTL, שפת העיצוב והטוקנים הקיימים, ונגישות (ARIA, focus, ניווט מקלדת
בתוך הקנבס). השתמש באותה גרסת @xyflow/react של הבלוק הקיים; הפעל את מצב
הקריאה-בלבד דרך props (nodesDraggable=false, nodesConnectable=false,
elementsSelectable מוגבל).
```

---

## 5. הערות

- **אותו data, שני מצבים:** עריכה (העורך המלא) מול תצוגת-לומד (read-only) —
  אותו רכיב טופולוגיה עם props שונים. אל תשכפל לוגיקה.
- **read-only ב-React Flow:** `nodesDraggable=false`, `nodesConnectable=false`,
  `panOnDrag`/`zoomOnScroll` פעילים, `fitView`.
- **ביצועים:** טופולוגיה גדולה בתוך שיעור נגלל — render רק כשנכנס ל-viewport
  (lazy/intersection observer).
- **מיגרציה:** הבלוק משתמש ב-data הקיים; שום שינוי סכמה.
