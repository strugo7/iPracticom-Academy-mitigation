# בריף ל-Claude Design — דף תוכן ההכשרה (TrackDetails)

> **מטרה:** לעצב את הדף הפנימי של מסלול הכשרה — ניווט בהיררכיית מודול → נושא → שיעור, עם סטטוס ביצוע לכל שיעור.
> **תפקיד הדף (מ-PRD §11):** `TrackDetails` — פירוט מסלול + התקדמות.
> **הפלט:** עיצוב ויזואלי → יוזן בהמשך ל-Claude Code למימוש מול Supabase.

---

## ההיררכיה (לפי SRS §1.2)

```
מסלול (LearningTrack)
  └── מודול (SharedModule) — שם, משך מוערך, תיאור
        └── נושא (Topic) — שם, תיאור, סדר
              └── שיעור (ModuleLesson) — שם, duration_minutes, xp_reward  ← הלב
                    └── מבחן מקושר (אופציונלי)
```

> "משך" של נושא לא קיים כשדה ישיר — נגזר מסכום `duration_minutes` של השיעורים שבו.

---

## כיוון העיצוב — תצוגת תוכנית-לימודים נפתחת (Accordion)

הדפוס המוכח ל-LMS: מודולים כסקשנים נפתחים, נושאים מקוננים, שיעורים כשורות.
למה זה הכי אינטואיטיבי כאן: משקף ויזואלית את ההיררכיה, מציג התקדמות בכל רמה,
ומתמודד עם נעילת רצף (`require_previous_lesson`). המודול הנוכחי נפתח אוטומטית.

**רכיבים לשימוש חוזר שייבנו:**
1. `TrackProgressHeader` — כותרת המסלול + התקדמות כללית.
2. `ModuleSection` — סקשן מודול נפתח.
3. `TopicGroup` — קבוצת נושא בתוך מודול.
4. `LessonRow` — שורת שיעור עם סטטוס (**הרכיב הקריטי**).

## 📋 הפרומפט — העתק מכאן

```
אני רוצה לעצב את דף תוכן ההכשרה (TrackDetails) של iPracticom Academy.
חשוב מאוד: השתמש ברכיבים, בטוקנים ובשפת העיצוב הקיימים בפרויקט (כולל
הדשבורד ודף ההכשרות) — עקביות מלאה, אל תיצור שפה חדשה.

— זהות מותג (לאכיפה) —
• עברית מלאה, RTL.
• צבעים: ראשי #2EB4FF, כחול כהה #0075DB, אפור #757D86, כהה #181D24,
  רקע בהיר #F4FBFF.
• שפה: tech-human, נקייה, אייקוני outline, אלמנטים גאומטריים, אנימציות עדינות.
• לוגו שמאל-עליון ב-50% שקיפות.
• פוטר: © iPracticom Academy - לשימוש פנימי בלבד

— מהות המסך —
דף פנימי של מסלול הכשרה. המשתמש נכנס למסלול ורואה את כל התוכן שלו מאורגן
בהיררכיה: מודולים → נושאים → שיעורים. המטרה: ניווט אינטואיטיבי, והבנה מיידית
של "איפה עצרתי ומה הלאה".

השתמש בתבנית תוכנית-לימודים נפתחת (accordion): מודולים כסקשנים נפתחים,
נושאים מקוננים בתוכם, שיעורים כשורות. המודול שבו המשתמש נמצא כרגע — פתוח
אוטומטית. בנה רכיבים לשימוש חוזר: TrackProgressHeader, ModuleSection,
TopicGroup, LessonRow.

— מבנה הדף —

1) כותרת מסלול (TrackProgressHeader):
   • שם המסלול + תמונת רקע/hero עדינה.
   • בר/טבעת התקדמות כללית באחוזים.
   • "X מתוך Y שיעורים הושלמו" + סך XP שנצבר במסלול.
   • כפתור ראשי "המשך מהמקום שעצרת" → השיעור הנוכחי.

2) מודול (ModuleSection — נפתח/נסגר):
   • כותרת המודול + משך מוערך + מונה נושאים/שיעורים.
   • בר התקדמות ברמת המודול (שיעורים שהושלמו במודול).
   • חץ נפתח. המודול הנוכחי פתוח כברירת מחדל.

3) נושא (TopicGroup — בתוך מודול):
   • כותרת הנושא + משך (סכום השיעורים) + מונה שיעורים.
   • רשימת שורות השיעור.

4) שורת שיעור (LessonRow) — הרכיב הקריטי, ארבעה מצבים:
   • שם השיעור + משך (duration_minutes).
   • בר התקדמות + אינדיקטור סטטוס:

     — טרם התחיל: בר על 0%, גוון נייטרלי/אפור, כפתור "התחל".
     — בתהליך: בר התקדמות חלקי בגוון #2EB4FF המשקף את הנקודה שבה עצר;
       לחיצה על "המשך" מחזירה אותו לאותה נקודה בדיוק.
     — הושלם: אינדיקטור ירוק + סימון V, ותגית XP שהרוויח (למשל "+10 XP").
       בר מלא בגוון ירוק.
     — נעול (אם השיעור הקודם חובה ולא הושלם): אייקון מנעול, השורה מעומעמת
       ולא לחיצה, כיתוב "השלם את השיעור הקודם כדי להמשיך".

   • בקצה הנושא/מודול — אם יש מבחן מקושר, הצג שורת "מבחן נושא/מודול" נפרדת.

— מצבים נדרשים —
• מצב מאוכלס (ברירת מחדל) — מסלול עם 2 מודולים, נתוני דמו בעברית (ראו למטה).
• מובייל — רספונסיבי; הסקשנים נשארים נפתחים-נסגרים בנוחות במגע.

— נתוני דמו (השתמש בהם) —
מסלול: "הכשרת טכנאי שטח".
מודול 1 "יסודות רשתות ותקשורת" (פתוח):
  נושא "מבוא לרשתות":
    • "מהי כתובת IP" — הושלם, +10 XP
    • "DHCP ו-DNS" — בתהליך, 40%
    • "מבוא ל-VLAN" — טרם התחיל
מודול 2 "MikroTik Firewall" (סגור):
  נושא "חוקי Firewall בסיסיים":
    • "מבנה טבלת ה-Filter" — נעול

— אילוצים טכניים —
React + Tailwind + shadcn/ui, אייקוני lucide-react (outline),
אנימציות עדינות (framer-motion). לאינדיקטור ההשלמה הירוק — השתמש בטוקן
ה-success הקיים במערכת; אם אין, בחר ירוק רענן שמתואם לפלטה.
שמור על רכיבים קיימים — אל תפשט/תמיר.

תתחיל ממצב המאוכלס בדסקטופ, ואחר כך נדייק.
```

## אנטומיית הרכיבים — רפרנס (שדות SRS)

| רמה | אלמנט | שדה נתונים (SRS) |
|---|---|---|
| מסלול | שם / תיאור / תמונה | `LearningTrack.title/description/image_url` |
| מסלול | התקדמות כללית | `progress_stats.avg_progress` |
| מודול | שם / משך | `SharedModule.title` / `estimated_duration` |
| מודול | סדר | `TrackModule.order_index` |
| נושא | שם / תיאור / סדר | `Topic.title/description/order_index` |
| נושא | משך | סכום `duration_minutes` של השיעורים |
| שיעור | שם | `ModuleLesson.title` |
| שיעור | משך | `ModuleLesson.duration_minutes` |
| שיעור | XP | `ModuleLesson.xp_reward` (ברירת מחדל 10) |
| שיעור | נעילת רצף | `ModuleLesson.require_previous_lesson` |
| שיעור | סטטוס/התקדמות | `UserProgress.progress_type` + `completion_percentage` |
| שיעור | מבחן מקושר | `ModuleLesson.linked_exam_id` |

> **מקור הסטטוס:** שיעור "הושלם" = קיימת `UserProgress` מסוג `lesson_completed`.
> "בתהליך" = `lesson_started` עם `completion_percentage` חלקי (זו גם נקודת ה"המשך").
> "טרם התחיל" = אין רשומה. "נעול" = `require_previous_lesson=true` והקודם לא הושלם.

## בקשת ה-handoff (לסיום)

```
מצוין. הכן סיכום מימוש למסירה למפתח:
• הרכיבים: TrackProgressHeader, ModuleSection, TopicGroup, LessonRow —
  עם ה-props שכל אחד מצפה לקבל (בשמות השדות שציינתי).
• ארבעת מצבי ה-LessonRow (טרם התחיל / בתהליך / הושלם / נעול) במפורש.
• טוקני העיצוב (צבעים כולל גוון ההצלחה הירוק, מרווחים, טיפוגרפיה, רדיוסים).
• התנהגות accordion (פתיחה אוטומטית של המודול הנוכחי) והערות מובייל.
שמור על React + Tailwind + shadcn.
```
