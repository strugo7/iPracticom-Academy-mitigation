# בריף עיצוב — מבחנים ומאגר שאלות

> **מטרה:** עיצוב מאגר השאלות (ישות עצמאית) ובונה המבחנים (הפניות many-to-many).
> **תפקיד:** חלק מ-`ContentManager` — instructor+.
> **מקורות:** SRS §1.4 (Question, Exam), §2.4 (importQuestionsCSV, quizMasterAgent).
> **רצף:** משטח 3 מתוך מפעל התוכן (מסמך 12).

---

## 1. המבנה הארגוני (קריטי להבנה)

- **שאלה (`Question`)** = ישות עצמאית במאגר. נוצרת פעם אחת, חיה במאגר.
- **מבחן (`Exam`)** = אוסף **הפניות**: `questions[] = {question_id, order_index, points}`.
  המבחן מצביע על שאלות, לא מכיל אותן.
- **many-to-many:** שאלה אחת → מופנית מכמה מבחנים. כל מבחן קובע לה סדר וניקוד משלו.
- **אובייקט משותף:** עריכת שאלה משפיעה על *כל* המבחנים שמפנים אליה
  (`usage_count` מראה כמה). דפוס אזהרה כמו SharedModule.

```
מאגר השאלות (Questions)          מבחנים (Exams)
┌──────────────┐                 ┌─────────────────────┐
│ Q1 ──────────┼──── question_id ┤ מבחן A: [Q1,Q3,Q7]   │
│ Q2           │            ╲    └─────────────────────┘
│ Q3 ──────────┼──────────── ╲── ┌─────────────────────┐
│ Q7 ──────────┼──────────────── ┤ מבחן B: [Q1,Q2]      │
└──────────────┘                 └─────────────────────┘
```

## 2. ארבעת סוגי השאלות

| סוג | מבנה עריכה |
|---|---|
| `multiple_choice` | נוסח + רשימת `options[]` (הוסף/הסר) + סימון הנכונה (`correct_answer_index`) |
| `true_false` | נוסח + מתג נכון/לא-נכון |
| `matching` | נוסח + זוגות התאמה (פריט ימני ↔ פריט שמאלי), הוסף/הסר |
| `order_sequence` | נוסח + פריטים (`order_items`) שמסדרים בסדר הנכון (גרירה) |

**שדות משותפים לכולם:** `category`, `topic_tags`, `difficulty_level`,
`explanation`, `points`, `status`(draft/published/archived).

## 📋 פרומפט A — מאגר השאלות + עורך השאלות (העתק מכאן)

```
אני רוצה לעצב את מאגר השאלות של iPracticom Academy ואת עורך השאלות.
השתמש בשפת העיצוב והטוקנים הקיימים — עקביות מלאה, RTL.

— מותג: עברית RTL · #2EB4FF / #0075DB / #757D86 / #181D24 / רקע #F4FBFF ·
  tech-human · אייקוני outline · אנימציות עדינות.

— מאגר השאלות (Question Bank) —
• כותרת "מאגר השאלות" + כפתור "שאלה חדשה" + ייבוא CSV.
• חיפוש + סינון: קטגוריה, סוג שאלה, רמת קושי, תגיות, סטטוס.
• רשימת/טבלת שאלות: נוסח (קצר), תגית סוג, קטגוריה, רמת קושי, ניקוד,
  שיעור הצלחה (success_rate), ומונה שימוש (usage_count = בכמה מבחנים).
• פעולות: ערוך, שכפל, מחק, שנה סטטוס.
• בעריכת שאלה בשימוש — אזהרה "השאלה מופיעה ב-N מבחנים; עריכה תשפיע על כולם".

— עורך השאלה (מודאל/drawer) — גוף מתאים לסוג —
• בורר סוג: רב-ברירה / נכון-לא נכון / התאמה / סידור.
• רב-ברירה: נוסח + רשימת תשובות (הוסף/הסר) + סימון הנכונה (רדיו).
• נכון/לא-נכון: נוסח + מתג נכון/לא-נכון.
• התאמה: נוסח + זוגות (ימני ↔ שמאלי), הוסף/הסר זוג.
• סידור: נוסח + פריטים הניתנים לגרירה לסדר הנכון.
• שדות משותפים: קטגוריה, תגיות, רמת קושי, ניקוד, הסבר (explanation), סטטוס.

— נתוני דמו (עברית) —
• "מהי כתובת ברירת המחדל של MikroTik?" (רב-ברירה, רשתות, 3 שימושים)
• "DHCP מקצה כתובות IP אוטומטית" (נכון/לא-נכון)
• "התאם פרוטוקול לפורט: HTTP↔80, HTTPS↔443, DNS↔53" (התאמה)
• "סדר את שלבי אתחול מצלמת IP" (סידור, 4 פריטים)

— מצבים: דסקטופ-first + מובייל. הצג גם את עורך השאלה פתוח על שאלת רב-ברירה.
— טכני: React + Tailwind + shadcn/ui, lucide-react (outline), dnd-kit (לסידור),
  framer-motion. שמור על רכיבים קיימים.

תתחיל מהמאגר המאוכלס בדסקטופ, ואז הצג את עורך השאלה.
```

## 📋 פרומפט B — בונה המבחנים (אחרי שהמאגר מוכן)

```
עכשיו עצב את בונה המבחנים (Exam Builder). אותה שפת עיצוב, RTL.

— מבנה הדף —
1) רשימת מבחנים: כותרת, סוג מבחן, מספר שאלות, ציון מעבר, סטטוס + "מבחן חדש".
2) בונה המבחן (נפתח ביצירה/עריכה) — שני אזורים:

   אזור פרטי המבחן (עליון):
   • כותרת, סוג מבחן (track/module/topic/lesson/standalone), מה הוא מקושר אליו,
     רמת קושי, ציון מעבר (passing_score), תגיות.
   • אם "מבחן כניסה" (is_entrance_exam): תפקידי-יעד ומחלקות-יעד.

   אזור השאלות (מרכזי) — לב הדף:
   • רשימת השאלות שנוספו למבחן, ניתנות לגרירה לסידור (order_index),
     כל אחת עם ניקוד הניתן לעריכה (points override) וכפתור הסרה.
   • כפתור בולט "הוסף שאלות" → פותח פאנל צד:

     פאנל הוספה (הלב של הזרימה):
     • חיפוש + סינון של מאגר השאלות (קטגוריה/סוג/קושי/תגיות).
     • כל תוצאה: תצוגה מקדימה + כפתור "הוסף" (מסומן אם כבר במבחן).
     • בחירה מרובה + "הוסף נבחרות".
     • כפתור בולט "צור שאלה חדשה" → פותח את עורך השאלה; בשמירה השאלה
       נוספת גם למאגר וגם למבחן הנוכחי בפעולה אחת.

3) תצוגה מקדימה של המבחן + שמירה/פרסום.

— נתוני דמו: מבחן "יסודות רשתות" (module_exam, ציון מעבר 70), 4 שאלות מהמאגר.
— מצבים: דסקטופ. הצג את פאנל ההוספה פתוח לצד המבחן.
— טכני: React + Tailwind + shadcn/ui, dnd-kit, lucide-react. שמור על רכיבים קיימים.

תתחיל מבונה המבחן עם 3 שאלות שנוספו ופאנל ההוספה פתוח.
```

## אנטומיה — רפרנס שדות (SRS §1.4)

| אלמנט | שדה |
|---|---|
| שאלה | `Question.{question_text, question_type, category, topic_tags, difficulty_level, options, correct_answer_index, order_items, explanation, points, status, usage_count, success_rate}` |
| מבחן | `Exam.{title, exam_type, is_entrance_exam, target_roles, target_departments, context_type, context_id, linked_*_id, passing_score, questions[]}` |
| הפניה במבחן | `questions[].{question_id, order_index, points}` |
| ייבוא/ייצוא | `importQuestionsCSV`, `exportQuestionsCsv`, `importExistingQuestions` |
| יצירת AI | `quizMasterAgent` (קורא תוכן שיעור → מייצר שאלות) |
